# Phase 08 — UI Design Contract: AI Debug

> Visual and interaction contract for the first user-invokable AI feature in v1.1. Generated by gsd-ui-researcher (auto mode), pre-populated from 08-CONTEXT.md, 08-RESEARCH.md, 07-CONTEXT.md (disclosure precedent), 06-CONTEXT.md (MarkdownRenderer.render precedent), and load-bearing existing files (`src/main/codeBlockButtonRow.ts`, `src/solve/VerdictModal.ts`, `src/ai/disclosure.ts`, `styles.css`).

| Property | Value |

### Button labels

| Element | Copy |

| Verdict modal footer button (conditional on non-Accepted) | `AI: Debug` |
| RunModal failure footer button | `AI: Debug` |

| AIStreamModal Close button (post-completion) | `Close` |

**Why `AI: Debug` not `AI Debug` or `Debug`:**

- Parity with palette label `AI: Debug current code` (consistent visual token "AI:" cues that this contacts an external AI provider, not a local feature).
- Distinguishes from any future non-AI debug surface.
- Locked by CONTEXT decision C; researcher recommendation in 08-CONTEXT `<open_questions_for_planning>` was `AI: Debug`; auto-mode confirms.

### Exceptions / Locked single-word labels

CONTEXT decision D locks two single-word button labels in the AIStreamModal footer:

- **`Cancel`** — mid-stream abort button (visible during streaming). Single word is intentional. Matches established Obsidian modal conventions for in-flight cancellation (`Modal` core uses `Cancel` as its standard short-label); pairing it with a longer phrase ("Cancel request" / "Stop streaming") would diverge from the pattern users learn from every other Obsidian dialog. Verb-only is the Obsidian house style.
- **`Close`** — post-stream / post-cancel / post-error dismiss button. Single word is intentional. Matches Obsidian's standard dismissal-button label across the app (Settings, callout dialogs, the disclosure modal's secondary action). A longer phrase ("Close window" / "Done") would create modal-to-modal inconsistency.

These two exceptions are the only single-word labels in Phase 08; every other user-visible button uses the longer prefixed form (`AI: Debug`, `Copy response`, `AI: Debug current code`).

### Body states

| State | Copy |
|-------|------|
| Initial (modal just opened, before disclosure resolves OR before first chunk arrives) | `Thinking…` (single ellipsis character `…`, U+2026 — NOT three dots) |
| Counter ticking (fallback path AND pre-first-chunk on stream path) | `Thinking… {mm}:{ss}` (e.g. `Thinking… 00:03`) — NBSP between `Thinking…` and digits |
| First chunk arrives (stream path) | Body cleared, counter stopped, MarkdownRenderer.render begins (debounced 100ms) |
| Stream complete | Final MarkdownRenderer.render flushed; footer becomes `[Copy response]` `[Close]` |
| Cancelled mid-stream | Insert `Cancelled — partial response below.` line ABOVE the existing rendered body; footer becomes `[Copy response]` `[Close]` |
| Cancelled before any chunk (fallback path) | Modal closes immediately (no in-modal end-state — partial output never existed) |
| Network/provider error | Replace body with `Couldn't reach {prettyName(provider)}.` heading + `<p>{vendor message — truncated to 200 chars}</p>`; footer becomes `[Close]` only |
| Empty-store path (LastVerdictStore.get returns undefined) | Modal opens normally; the prompt sent to AI contains the literal line `No verdict yet — review the code as-is.` (this is prompt text, NOT user-visible UI copy). User sees normal stream / Thinking state. |

**Verbatim error copy (locked):**

- Network reach failure heading: `Couldn't reach {prettyName(provider)}.` (apostrophe via U+2019 right-single-quote per project copy convention; matches Phase 06 `Couldn't load problem` at `ProblemPreviewView.ts:65`).
- Network reach failure body: `{vendor message}` truncated to 200 chars combined (mirrors Phase 07 D-04 truncation rule from 07-CONTEXT decision E).
- Disclosure-cancelled body (when AIClient.invoke throws `Error('AI call cancelled')` from the disclosure gate): `AI call cancelled.` — matches Plan 07-05 verbatim string (locked).

### Disclosure copy extension (composition rule)

Phase 08 appends ONE feature-specific bullet to `DISCLOSURE_BASE_COPY.willSend`. The composition seam is `withDebugBullet(base): DisclosureCopy` — exported from `src/ai/disclosure.ts` (recommended location; planner has final say).

```
AI Debug also sends the last failing run/submit verdict for this problem (input, expected output, your output, error message)
```

Verbatim. export function withDebugBullet(

## Behavior Contract

### Open path

1. User clicks any of the 3 surfaces (fence-row button / verdict-modal footer / palette command).
2. `LeetCodePlugin.openAIDebug(slug)` resolves prompt ingredients (problem markdown via `getProblemDetail`, `## Code` content via `extractFirstFencedBlock`, language from fence info-string, `LastVerdictStore.get(slug)` for verdict).
3. `buildDebugPrompt(...)` produces the prompt string (per 08-RESEARCH Pattern 3).
4. `new AIStreamModal({ provider, prompt, aiClient, disclosureCopy: withDebugBullet(DISCLOSURE_BASE_COPY) }).open()`.
5. Modal `onOpen()` paints `Thinking…` placeholder, starts mm:ss counter, calls `aiClient.invokeStream(req)`.
6. AIClient gate fires `requireDisclosure(provider, cfg)` if `cfg.disclosureAcknowledged === false` — disclosure modal opens on top.
7. On Continue: persist `disclosureAcknowledged: true`, return; AIClient resolves adapter; `streamText({ model, prompt, abortSignal })` begins.
8. First chunk → body `.empty()`, counter `stopCounter()`, MarkdownRenderer.render begins (debounced 100ms ring buffer per 08-RESEARCH Pitfall 1 Tier 1).

### Cancel paths (AIDBG-03 lock)

| Path | Trigger | User-visible result |
|------|---------|---------------------|
| Stream-time Cancel button click | User clicks `[Cancel]` mid-stream | `abortController.abort()` fires; for-await iterator throws; `signal.aborted === true` short-circuit → footer becomes `[Copy response]` `[Close]`; "Cancelled — partial response below." line prepended; cost ledger adds 0 (08-RESEARCH Pitfall 6) |
| Stream-time modal dismiss (Esc / X / overlay-click) | User dismisses while streaming | `onClose()` calls `abortController.abort()` + clears `renderTimer` + clears `counterTimer` + empties `contentEl`; cost ledger adds 0 |
| Disclosure-time Cancel button | User cancels in disclosure modal | AIClient throws `Error('AI call cancelled')`; AIStreamModal catches, body shows `AI call cancelled.`, footer becomes `[Close]` only |
| Disclosure-time dismiss (Esc / X / overlay-click) | User dismisses disclosure modal without choosing | Same as Cancel (Phase 07 Plan 05 lock — `decided`/`acknowledged` flag posture in `disclosure.ts:172-189`) |
| Fallback path Cancel (requestUrl, no abort support) | User clicks `[Cancel]` while `Thinking…` is showing on fallback path | Modal closes immediately (`this.close()`); in-flight requestUrl call may complete in background; AIStreamModal's resolver is wrapped in `.catch(() => {})` (08-RESEARCH Pitfall 3) — no unhandled rejection. No partial output exists to preserve. |

### Render path

- **Stream available** (`electron.net.fetch` resolves; `obsidianFetch('stream')` returns streaming Response):
  - Body shows `Thinking… 00:00` until first chunk.
  - First chunk arrives: counter stops, body cleared, debounced MarkdownRenderer.render begins.
  - Each subsequent chunk appends to internal buffer; render scheduled via `setWindowTimeout(..., 100)` ring buffer; only ONE render outstanding at a time.
  - Stream-end: final flush render, footer swap to `[Copy response]` `[Close]`, cost ledger updated via `onFinish({ totalUsage })` event.
- **Fallback path** (`loadElectronNet` throws → `obsidianFetch('request')` + `generateText`):
  - Body shows `Thinking… 00:00`; counter ticks every 1000ms via `setWindowTimeout` recursion (per `src/shared/timers.ts` precedent).
  - On response arrival (single buffered chunk): counter stops, body cleared, MarkdownRenderer.render runs ONCE on the full text, footer swap.
  - Counter clamps at `99:59+` (defensive — should never actually hit this).

### Copy → clipboard (post-stream / post-cancel)

- Click `[Copy response]` → `navigator.clipboard.writeText(this.buffer)` → Notice `AI response copied.` (2000ms).
- On clipboard exception: Notice `Clipboard unavailable.` (4000ms). No fallback prompt — user can select-all-copy from the rendered Markdown body.

### Cleanup (locked)

`AIStreamModal.onClose()` MUST:

1. Abort `abortController` if `!completed && !cancelled` (anti-zombie per AIDBG-03 + 08-RESEARCH Pitfall 4).
2. `clearWindowTimeout(this.renderTimer)` if non-null.
3. `clearWindowTimeout(this.counterTimer)` if non-null.
4. `this.contentEl.empty()`.

No `unregisterEvent`, no DOM listener cleanup beyond `contentEl.empty()` (button listeners ride along with the DOM nodes).
