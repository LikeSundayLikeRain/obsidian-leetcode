## Accomplishments

- C6 Esc: doc-level capture-phase keydown listener attached/removed in lockstep with the dropdown open/close lifecycle. Esc now closes the dropdown regardless of focus location (button, item, or anywhere else).
- G-CLICK-THROUGH: single wrapper-level pointerdown stopPropagation. CM6's caret-positioning runs on pointerdown (before click), so the previous click-time mitigation arrived too late; the new wrapper-level listener catches both the chevron button AND every dropdown item with one handler.
- G-LABEL-LAG: exported `languageRefreshEffect` (StateEffect<void>); widened the StateField update predicate to rebuild on it; subscribed `buildCodeActionsEditorExtension` to `metadataCache.on('changed')` and dispatch the effect on the active MarkdownView's EditorView when the changed file matches. Chevron label now refreshes synchronously on the same click as the fence/frontmatter writes.
- G-CHEVRON-STYLING: ported `.lc-fm__picker` block to `.leetcode-language-chevron` — `min-height: 30px`, `padding: 2px 8px`, `font-size: 13px`, `background: var(--background-primary)`, `border-radius: 6px`, `transition: border-color 120ms ease`; hover changes border-color to `var(--text-muted)` (no more background/color flip).
- G-LAYOUT: replaced the hasLineAfterCloser conditional with a single canonical anchor — `state.doc.line(fence.closerLine).to` + `side: 1`. The closer-fence line's indent is always the fence's own (0 for top-level `## Code` blocks), so typing Tab on the post-closer line no longer drifts the action row.
- G-PYTHON-LABEL: `python3 → 'Python 3'` (was `'Python'`) in `LC_LANG_DISPLAY_LABELS`. The chevron now signals which Python the LC API call will dispatch; `python` (LC's deprecated Python 2) still maps to `'Python'` — harmless because rarely selected.

## Decisions Made

- **G-LABEL-LAG path choice (PATH A vs PATH B).** Chose PATH A: the editor extension owns the `metadataCache.on('changed')` subscription and dispatches `languageRefreshEffect` on the active MarkdownView's EditorView. Rationale: (a) keeps the chevron-freshness contract local to the file that owns chevron rendering, (b) `src/main.ts` stays unaware of the StateField's rebuild contract, (c) PATH A also closes the secondary refresh gap — when the user edits `lc-language` directly via Obsidian's property panel (bypassing `switchFenceLanguage`), the chevron still refreshes. PATH B (caller-driven dispatch from `switchFenceLanguage`) was viable but would have required adding a parallel notify path for the property-panel edit case.
- **G-CLICK-THROUGH wrapper-level vs per-item.** Chose wrapper-level pointerdown (one listener catches chevron button + all 8 items) over per-item pointerdown handlers. Less code, single source of truth for the propagation contract, and the wrapper is the right scope because the entire chevron+dropdown surface is what must NOT bleed to CM6.
- **G-LAYOUT single anchor (no hasLineAfterCloser conditional).** Removed the end-of-doc fallback variable: `state.doc.line(closerLine).to` is a valid offset whether or not a post-closer line exists. The two-branch structure was a false dichotomy — both branches are now collapsed into the load-bearing one (the previous fallback path).
- **G-PYTHON-LABEL only flip python3.** Kept `python` as `'Python'` rather than renaming both (e.g., to `'Python 2'` and `'Python 3'`). LC's `python` slug is deprecated and almost never selected; flipping its label would invent a new visual distinction users haven't asked for. The disambiguation is one-sided because that's what the user feedback flagged.
