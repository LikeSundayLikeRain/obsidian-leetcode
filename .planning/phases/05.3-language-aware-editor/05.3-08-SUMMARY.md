# Phase 05.3 Plan 08: G-DROPDOWN-CLIPPED Gap-Closure Summary

**Chevron dropdown portaled to document.body so items are hit-testable outside CM6's cm-content `contain: paint` container — restoring the load-bearing language-switching UX.**

## Accomplishments

- Closed G-DROPDOWN-CLIPPED root cause (cm-content `contain: paint` clipping + breaking hit-test on overflowing descendants regardless of z-index/isolation)
- Portal pattern: dropdown is `doc.body.appendChild`'d on open, `dropdown.remove()`'d on close
- Position: `position: fixed` + `top: rect.bottom + 4px` + `left: rect.left` from `button.getBoundingClientRect()`
- Window `scroll` (capture phase) + `resize` listeners reposition while open; both detach on close (no leaks)
- Outside-click guard now checks BOTH `wrapper.contains(target)` AND `dropdown.contains(target)`
- pointerdown stopPropagation extended to dropdown (parity with wrapper-level G-CLICK-THROUGH guard, restoring caret-bleed prevention for body-portaled items)
- 6 new test it-blocks under `G-DROPDOWN-CLIPPED: portal pattern`; existing Esc / click-through / dropdown-toggle tests migrated to document-level dropdown query

## Decisions Made

See frontmatter `key-decisions`. Most load-bearing:
1. Portal-to-body over fighting `contain: paint` (Plan 07's z-index/isolation approach was already proven inert).
2. Native addEventListener (open-close-cycle scoped) vs plugin.registerDomEvent (plugin-lifetime scoped) — the former is correct for this lifecycle.
3. Capture-phase scroll on window, so nested scroll containers fire the listener (scroll does NOT bubble).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Restored G-CLICK-THROUGH coverage for body-portaled dropdown items**
- **Found during:** Task 5 (test run after refactor)
- **Issue:** Plan 05 attached a single `pointerdown` capture listener on `wrapper` to catch CM6 caret-bleed for both the chevron button AND every dropdown item. After Task 1's portal pattern, dropdown items are no longer descendants of `wrapper`, so the wrapper-level listener cannot intercept their pointerdown events. Test `pointerdown on dropdown item does NOT propagate to document` failed.
- **Fix:** Added a parallel `pointerdown` listener on the `dropdown` element with the same handler (`stopPointerdown`), reusing the existing handler reference. Updated the comment block to explain why both wrapper AND dropdown need the listener under the portal pattern.
- **Files modified:** `src/main/languageChevronWidget.ts`
- **Verification:** All 25 chevron tests passing; G-CLICK-THROUGH coverage restored.
- **Committed in:** `c8c86c7` (Task 5 commit, alongside the test updates).
