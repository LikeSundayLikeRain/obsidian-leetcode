<truths>
Locked CONTEXT decisions this plan implements:
- **D-05:** Single-transaction CM6 dispatch carrying fence opener + fence body. `processFrontMatter` writes `lc-language` separately (vault undo stack — divergence accepted per Pitfall 1).
- **D-06:** Chevron LEFT-aligned in Phase 5.1's row. Layout `[▼ Python]                  [Run] [Submit]` via `justify-content: space-between` + `margin-left: auto` on the run button.
- **D-07:** Always visible (Phase 5.1 parity). Chevron renders on every lc-slug note that has a `## Code` fence regardless of cursor position. Same gate as `buildDecorations` already enforces.
- **D-08:** Destructive replace on switch. No confirm dialog. Cmd-Z reverts the fence atomically; vault frontmatter undoes separately (two-Cmd-Z divergence accepted).
- **D-09:** Edit-Mode only — no Reading-Mode chevron. `codeActionsPostProcessor.ts` is UNCHANGED — must NOT pass `opts.prefix`.
- **D-10:** Chevron text = `▼ {LC_LANG_DISPLAY_LABELS[currentSlug] ?? currentSlug}`.
- **D-11:** Refetch starter code from LC GraphQL on every switch. Existing `LeetCodeClient.getProblemDetail(slug)` already wraps `requestUrl` + 7-day cache + throttle. Network failure → silent fallback (keep current body) + ONE Notice copy: `Couldn't fetch starter code for {Label}.` (UI-SPEC §"Error state — network failure on starter-code refetch").
- **D-12:** No reverse-mapping. Plugin reads `lc-language` frontmatter for Run/Submit dispatch — never the fence tag.

Per UI-SPEC §"Dropdown item click → language switch" — the order is load-bearing:
1. Step A — fetch starter code via `plugin.client.getProblemDetail(slug)`.
2. Step B — single CM6 `view.dispatch({ changes: [openerChange, bodyChange], userEvent: 'leetcode.lang-switch' })`.
3. Step C — `await app.fileManager.processFrontMatter(file, fm => { fm['lc-language'] = newSlug; })`.

Reasoning (RESEARCH §Pattern 4 + Anti-Pattern): processFrontMatter is async and would delay user-visible fence rewrite by 5–20 ms. Doing it AFTER CM6 dispatch keeps the visible change instant. Doing it BEFORE creates a window where `lc-language` says Java but fence still has Python → Run during that window dispatches mismatched language to LC.

Per UI-SPEC §"Coexistence with Phase 5.1 Run/Submit buttons" — `CodeActionsWidget.eq()` MUST account for the current language so the widget rebuilds when `lc-language` flips (otherwise the chevron label stays stale until the user types something to trigger `docChanged`). Capture `slug` and current `lc-language` at widget construction in `buildDecorations` and include them in `eq()`.

Per UI-SPEC §"Bundle-Size Gate" — recommended cap revision is 250 KB hard / 200 KB warn, but this plan does NOT change `scripts/check-bundle-size.sh`. Wave 3 (live-smoke + ship gate) ships the cap tightening alongside its bundle-size verification.
</truths>
