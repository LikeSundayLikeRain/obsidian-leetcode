## Gaps

- **C6 — Esc key does not close the dropdown.** UI-SPEC §Accessibility lists Esc dismissal as in-scope (basic a11y, not stretch). Likely missing `keydown` listener for `Escape` in `languageChevronWidget.ts`. Small fix; candidate for gap-closure plan.

- **G-LAYOUT — Chevron button shifts horizontally when text outside the fence is indented.** Observed: typing Tab on the line directly below the button row visibly moves the button along with the indentation. Indicates the action row is rendered as a CM6 line widget that respects indent guides of nearby lines, instead of being anchored to the fence container. Cosmetic but visible. Likely fix: render the button row as a block widget anchored to the fence end, not as a line decoration on the post-fence line. Reading-Mode postprocessor (D-09 — unchanged) is unaffected.

- **G-LABEL-LAG — Chevron label shows previous selection (one click stale).** Observed: switching python → java → c, the chevron text shows `▼ Python` → `▼ Python` → `▼ Java` — label lags one click behind. **Critical: the fence body, opener tag, and `lc-language` frontmatter ALL update synchronously on the first click — D-08 atomicity holds.** Only the chevron's own label re-render is stale. Root cause likely: the widget reads `lc-language` from Obsidian's metadataCache *before* the cache observes the frontmatter write (Pitfall 3 — cold/stale metadataCache, in the refresh-after-write direction rather than initial-render). Likely fix: the `switchFenceLanguage` handler should also force the chevron widget to refresh from the new slug it just wrote, instead of waiting for the metadataCache → buildDecorations → eq() pipeline to round-trip. Small fix; user-explicitly requested.

- **G-UNDO-ORDER — Cmd-Z undo order is reversed from spec (acceptable, observational only).** Spec (D-08): 1st Cmd-Z reverts visible fence (opener+body), 2nd Cmd-Z reverts vault frontmatter. Observed: 1st Cmd-Z reverts vault frontmatter (`lc-language` flips back), 2nd Cmd-Z reverts fence opener+body atomically. Two Cmd-Z's total — total step count matches spec. End-state correct. Each revert is internally atomic (D-08 atomicity within CM6 stack preserved). User accepts the divergence; flagged because it diverges from CONTEXT.md Pitfall 1's documented order assumption. NON-BLOCKING.

- **G-CLICK-THROUGH — Dropdown clicks propagate to underlying editor.** Observed: clicking a language item in the dropdown also moves the editor caret to the underlying click position. The dropdown click handlers are missing `event.stopPropagation()` (or are not in a layer that catches the click before CM6's caret-positioning handler). Likely fix: add `pointerdown`/`mousedown` `stopPropagation` on the dropdown popover root, or wrap the dropdown in `cm-no-edit` / use `pointer-events` discipline. Small fix; user-visible.

- **G-PYTHON-LABEL — Python display label inconsistent with stored slug; LC supports both `python` and `python3`.** Observed: chevron labels Python as `Python` but the underlying LC slug stored in `lc-language` frontmatter is `python3`. LC's API supports BOTH `python` (Python 2 — deprecated but still accepted) and `python3` (mainline). Two viable resolutions: (A) Add `python` entry to `LC_CHEVRON_LANG_ORDER` (9 items total) and rename `python3` label to `Python 3`; (B) Keep 8 items but rename the existing `Python` label to `Python 3` since Python 2 is rarely used. LOW priority, cosmetic only — non-blocking ship.

- **G-COPY-TO-CODE-LANG-DRIFT — Retrieving a previous submission via Copy-to-Code does not update `lc-language` or chevron.** Observed: copying a stored submission whose language differs from the note's current `lc-language` overwrites the fence body but leaves `lc-language` frontmatter and the chevron label unchanged. Result: the note's stored language and the actual code in the fence drift apart; subsequent Run/Submit will dispatch with the wrong slug. Likely fix: the copy-to-code handler in `src/solve/SubmissionDetailModal.ts` (or wherever copy-to-code is wired) should detect the submission's language slug, write `lc-language` via `processFrontMatter`, and rewrite the fence opener tag in the same dispatch — same atomic shape as `switchFenceLanguage`. Phase 5.3-adjacent integration gap; not directly in 5.3's locked scope but surfaced by the chevron+frontmatter-as-source-of-truth model. Recommend a separate small phase or gap-closure plan.

- **G-CHEVRON-STYLING — Chevron button styled as outlined button; user expected `.lc-fm__picker`-style input-field look (consistency with browser's filter dropdown).** Observed: current chevron renders with the secondary-background button look from UI-SPEC. User feedback: prefers the cleaner picker style used in `.lc-filter-modal .lc-fm__picker` (min-height 30px, 1px `var(--background-modifier-border)` border, 6px border-radius, `var(--background-primary)` background, hover border-color → `var(--text-muted)`). Recommend porting `.lc-fm__picker` styling to `.leetcode-language-chevron` for cross-feature visual consistency. Cosmetic; non-blocking ship.
