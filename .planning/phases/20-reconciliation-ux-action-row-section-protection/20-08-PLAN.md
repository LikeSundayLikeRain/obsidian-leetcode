<objective>
Fix gap 2 (language switch produces zero visible feedback — `20-HUMAN-UAT.md` Test 3,
severity major).

Symptom: user reports "Theme retheme looks good for Java (current language), but the
language switch is broken — looks like it hasn't been wired up. Couldn't verify all 8
language packs."

Confirmed root cause (`.planning/debug/language-switch-not-wired.md`): the chevron click IS
wired end-to-end (chevron → `widgetActions.ts:96-97` host adapter →
`switchLanguageFromWidget` → `processFrontMatter` → `metadataCache 'changed'` listener →
`languageCompartment.reconfigure`). But the click produces ZERO visible feedback because:
  1. Chevron LABEL is built once at `languageChevronWidget.ts:96` and never refreshed.
  2. Dropdown `.is-current` marker (`languageChevronWidget.ts:240-242`) is set once at
     build time and never refreshed.
  3. v1.3 deliberately does NOT swap fence body content (`switchLanguageFromWidget` only
     writes frontmatter — designed behavior per CONTEXT carry-over).
  4. `WidgetController` metadataCache listener (`WidgetController.ts:935-958`) reconfigures
     the parser only — has no handle to action-row DOM.

Net effect: silent frontmatter write → silent parser reconfigure → chevron still says "Java"
forever → indistinguishable from a no-op. The fix is path (a) from UAT.md — surgical DOM
update via an updater closure stored on the controller.

The fix introduces a CONTRACT: `buildLanguageChevron` returns a struct exposing the DOM
handles needed for refresh (labelSpan, dropdown item list). `mountActionRow` consumes those
handles and returns a single closure `(newSlug: string) => void` that updates labelSpan
text + re-targets the `.is-current` marker. `WidgetController.mountLeetCodeWidget` stores
the closure as `ctl.actionRowRefresh`. The existing metadataCache 'changed' listener inside
`mountLeetCodeWidget` is extended to call `ctl.actionRowRefresh?.(newSlug)` after the
existing Compartment.reconfigure dispatch.

**Plan 20-08 revision Blocker #3 — return-type change is breaking; test callers must be
updated explicitly.** The signature change from `HTMLElement` to
`{ wrapper, labelSpan, items, refresh }` is type-level breaking. The grep results before
this revision (run `grep -RnE 'buildLanguageChevron' src/ tests/`) show:
  - **27 production-test callers** in `tests/main/languageChevronWidget.test.ts` (lines
    75, 88, 99, 110, 121, 130, 141, 161, 180, 205, 221, 236, 257, 280, 306, 329, 355, 392,
    421, 437, 459, 487, 520, 554 — re-run grep before edit to confirm) all bind the result
    to `const wrapper = buildLanguageChevron(...)` and then call `.classList`,
    `.querySelector`, etc. on `wrapper`. Each of these MUST be rewritten to either
    (a) destructure: `const { wrapper } = buildLanguageChevron(...)` (preferred — minimal
    churn), OR (b) bind `.wrapper` explicitly:
    `const wrapper = buildLanguageChevron(...).wrapper`. Pattern (a) is simpler.
  - **ONE production caller** in `src/widget/widgetActions.ts` (the live mount path inside
    `mountActionRow`'s prefix factory). Task 2 of this plan rewrites this caller to capture
    the full handle.
  - **ONE production caller** in `src/main/codeActionsEditorExtension.ts:141` (the v1.2
    action-row mount path). This file is registered at `src/main.ts:894` via

This fix is purely DOM-additive in functional behavior — no API breakage in semantics

       let mountedSlug = currentSlug;
       const refresh = (newSlug: string): void => {
         if (newSlug === mountedSlug) return;
         mountedSlug = newSlug;
         // Update label text — fall back to the literal slug if no display

         // `languageChevronWidget.ts:96` and the resolveLanguageSlug behavior).

         // previous refresh raced) then add to the matching item if present.
         for (const item of items.values()) {

         if (target) target.classList.add('is-current');
         // Unknown slug: no item gets `.is-current`, label shows the literal
         // slug — same fallback shape as build-time unknown-slug handling.
       };
