<objective>
Fix gap 1 (atomicRanges cursor-edge cases — `20-HUMAN-UAT.md` Test 2, severity major).

Two coupled defects produce the symptom "for cases (a) and (b), it actually turns into source
mode instead of jump over" (`.planning/debug/atomicranges-cursor-edge-cases.md`):

  PRIMARY (deterministic for case b): `src/main/sectionProtectionExtension.ts:163-167`
  computes the `## Code` heading lock with upper bound `openerTo = state.doc.line(fence.openerLine + 1).from`.
  The transactionFilter at line 438-505 snaps any cursor entering `[headFrom, openerTo]` to
  `lockTo === openerTo` — which is the FIRST CHARACTER OF FENCE BODY LINE 1 and lies INSIDE
  the widget's `Decoration.replace([openerLine.from, closerLine.to])` range from
  `liveModeViewPlugin.ts:67-90`. The transactionFilter sets the cursor authoritatively
  bypassing atomicRanges. Live Preview's selection-overlap detector then strips the widget
  and reveals raw source.

  SECONDARY (intermittent for case a): `src/widget/liveModeViewPlugin.ts:84-89` calls
  `Decoration.replace({ widget })` without `block: true`. For a multi-line span CM6 treats
  this as inline replacement; vertical line motion via `cursorLineUp` (which uses
  `coordsAtPos`) can land at column-aligned positions inside the range. atomicRanges covers
  `cursorCharLeft/Right` motion but does not consistently cover screen-coordinate-based
  vertical motion through a multi-line inline replace.

Both fixes are surgical and mutually reinforcing:
  1. Widen the `## Code` snap target to land PAST the widget closer (so the
     transactionFilter never authoritatively sets the cursor inside the widget range).
  2. Add `block: true` to the Decoration.replace (so vertical motion treats the widget as
     a single visual block).

The plan is `autonomous: false` because verification is dev-vault-only (atomicRanges
behavior is runtime-only CM6, untestable under vitest per `20-VERIFICATION.md:121-129`).

Purpose: Phase 20 Test 2 passes — cursor jumps over widget body without revealing source.
Output: Two surgical edits + a manual UAT checkpoint.
</objective>
