# Phase 20 вҖ” UI Design Contract

> Visual and interaction contract for Phase 20 (reconciliation, in-widget action row, conflict modal, multi-pane affordance, live theme retheme). Design tokens are Obsidian's CSS variables (the platform-mandated palette for community plugins) вҖ” there is no custom palette.

| Property | Value |
|----------|-------|
| Tool | none (Obsidian plugin вҖ” shadcn does not apply) |

## Color

Obsidian CSS variables are the design tokens. **No raw hex / no rgba literals** in Phase 20 surfaces вҖ” verified against existing styles.css (only documented exceptions are the AI-Solution gradient stops `#7c3aed` вҶ’ `#4f46e5` and a single `rgba(0, 0, 0, 0.15)` popover-shadow precedent, neither of which Phase 20 introduces or modifies).

| Role | Value | Usage in Phase 20 |
|------|-------|-------------------|
| Dominant (60%) | `var(--background-primary)` | Modal body background; widget container body; chevron dropdown popover background; multi-pane greyed-out base layer |
| Secondary (30%) | `var(--background-secondary)` | Diff column `<pre>` panels (Mine / External / Merged); section dividers; chevron button hover state (when paired with `--background-modifier-hover`) |
| Accent (10%) | `var(--interactive-accent)` + `var(--text-on-accent)` | **Submit button** (existing rule, preserved); **modal "Keep mine" primary CTA** (`new Setting(...).setCta()`); chevron dropdown `.is-current` text via `var(--text-accent)` |
| Destructive | `var(--text-error)` + `var(--background-modifier-error)` | Diff "external-only" rows (lines that exist on disk but not in widget вҖ” they will overwrite Mine on "Keep external"); diff conflict marker gutter; multi-pane greyed-out warning ring (1px solid `var(--text-error)`, optional вҖ” `var(--text-muted)` is the lighter default) |
| Muted | `var(--text-muted)` + `var(--background-modifier-border)` | Multi-pane greyed-out widget overlay backdrop; "View diff" tertiary button border; diff "same" rows (no emphasis); modal body paragraph; AI-Solution gradient is preserved verbatim and is the documented exception |

**Accent reserved for (explicit list вҖ” never "all interactive elements"):**

1. **Submit button** in the action row (`.leetcode-code-action-submit` background `var(--interactive-accent)` вҖ” preserved verbatim from v1.2).
2. **AI Solution button** is a documented gradient exception (`#7c3aed вҶ’ #4f46e5`); it does NOT use `var(--interactive-accent)`. This is intentional вҖ” the gradient signals "AI" semantically across all themes. Preserved verbatim.
3. **"Keep mine" primary CTA** in the conflict modal (use Obsidian `Setting().addButton(b => b.setCta())` вҖ” Obsidian renders it with `var(--interactive-accent)` automatically).
4. **Chevron dropdown's currently-selected item** (`text-accent` color + weight 600 вҖ” preserved verbatim).
5. **Multi-pane "Click to take over" CTA** when the user hovers the greyed-out widget overlay (border becomes `var(--interactive-accent)`; text remains `var(--text-normal)`).

**Diff color contract (3-pane LCS diff inside conflict modal вҖ” Plan 20-03):**

**Multi-pane greyed-out contract (Plan 20-04, "Take over" CTA вҖ” option 1 from CONTEXT discretion):**

- Active (focused) pane has no overlay (default Phase 19 widget render).
- Locked: option 1 from RESEARCH.md В§"Pattern 8" (greyed-out + CTA). Options 2 (frozen-readonly) and 3 (banner-across-both) are rejected.

### 2. Conflict Modal вҖ” Inline Expansion Flow (SYNC-05, D-conflict-01..04)

State machine:

```
[State A: Modal opens]
  Heading: "External edit detected"
  Body: 1 paragraph + 3 buttons (Keep mine [CTA] / Keep external / View diff)

[State B: User clicks "View diff"]
  Same modal вҖ” appends diff container BELOW the buttons (does NOT replace them)
  3-column flex layout: Mine | External | Merged preview
  Buttons remain at top; user can still pick Keep mine / Keep external

[Terminal A: Click "Keep mine"]
  вҶ’ widget.writer.forceFlush() (immediate disk write)
  вҶ’ modal.close()
  вҶ’ toast Notice 3000ms: "Local edits saved." (NEW copy вҖ” see below)

[Terminal B: Click "Keep external"]
  вҶ’ widget.reloadFromDisk('keep-external')
  вҶ’ modal.close()
  вҶ’ toast Notice 3000ms: "Reloaded from disk." (NEW copy вҖ” see below)
```

Toast copy added to copywriting contract:
- Keep mine success: `Local edits saved.`
- Keep external success: `Reloaded from disk.`

D-conflict-04 (second external edit while modal open):

- `vault.on('modify')` handler checks `if (conflictModal.isOpen) conflictModal.updateExternalContent(newContent)`.
- External column re-renders silently. No banner. The 200ms `outline: 1px solid var(--text-muted)` flash is the only visual.

Modal sizing:

- Width: Obsidian default modal (`min(700px, 90vw)`). Do not override.
- Diff columns when expanded: `display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 12px;`. Each column `<pre>` has `max-height: 360px; overflow: auto;` matching `.leetcode-submissions-detail .leetcode-submissions-code` precedent.
- Horizontal scroll fallback (per RESEARCH A5): if pane < 600px, `grid-template-columns: 1fr` (stack vertically) вҖ” 3 stacked panes, each with their own scroll. Tested in Plan 20-03 against narrow Obsidian sidebar widths.

Syntax highlighting on diff Mine/External columns: **deferred** per CONTEXT discretion. v1 ships plain monospace (`var(--font-monospace)` 12px, `line-height: 1.6`). Plan 20-03 may add Lezer-based highlighting via `highlightTree` if budget permits вҖ” non-blocking.

Scroll synchronization: **synchronous via `scroll` listener on Mine pane mirroring to External + Merged**. ~10 LOC. Plan 20-03 ships this; if it feels janky in UAT, falls back to independent scroll per column.

### 6. Vim Live-Reconfigure (VIM-02) вҖ” No Visible UI Element

## Interaction Contracts

### Action Button Click

Inherited verbatim from `codeBlockButtonRow.ts:59,81`:

1. `mousedown` fires вҶ’ handler calls `e.preventDefault()` (focus retention вҖ” keeps the widget editor focused so subsequent typing lands in the doc).
2. `click` fires вҶ’ handler calls `e.preventDefault(); e.stopPropagation();` then dispatches the corresponding host method (`runFromActive` вҶ’ routed to `runFromWidget(ctl)` via the adapter).
3. The `*FromWidget` method:
   a. Calls `await widget.flushNow()` (forces any debounced disk write to complete).
   b. Reads `code = widget.view.state.doc.toString()`.
   c. Reads `lc-slug` + `lc-language` from `metadataCache.getFileCache(file).frontmatter`.
   d. Routes to the same downstream LC API call as `*FromActive`.

ARIA: action buttons inherit `<button>` semantics. Icon-only buttons use `title=` attribute (preserved verbatim) вҖ” no separate `aria-label`. The Phase 20 row does not add `role="toolbar"` вҖ” keep DOM minimal per Phase 5.3 D-09 lineage.

### Chevron Click вҶ’ Language Switch (ACTION-02, ACTION-03)

Inherited verbatim from `languageChevronWidget.ts:243вҖ“256` + new wiring:

1. User clicks chevron button вҶ’ `openDropdown()` вҶ’ dropdown portal-attaches to `doc.body` (G-DROPDOWN-CLIPPED), `position: fixed` anchored to button rect.
2. User clicks a different-language item вҶ’ `closeDropdown()` first (instant tactile feedback) вҶ’ `void plugin.switchLanguage(file, slug)` вҶ’ routes to NEW `switchLanguageFromWidget(ctl, file, newSlug)` (Plan 20-02).
3. `switchLanguageFromWidget`:
   a. `await ctl.flushNow()` (so any pending characters land under the OLD language slug).
   b. `await app.fileManager.processFrontMatter(file, fm => { fm['lc-language'] = newSlug; })`.
4. Obsidian fires `metadataCache.on('changed', file, ...)` automatically.
5. The widget's per-instance subscription handler:
   a. Re-reads `lc-language` from frontmatter cache.
   b. Builds new extensions: `buildLanguageExtensions(newSlug, indent)`.
   c. Dispatches: `widget.view.dispatch({ effects: languageCompartment.reconfigure(...) })`.
6. CM6 swaps language pack live. Cursor + scroll + undo preserved.

User-visible feedback: dropdown closes; chevron label updates to new language; syntax highlighting changes within ~16ms. No toast/Notice.

### External Edit Reconciliation (SYNC-04, D-conflict-03)

Decision tree (verbatim from RESEARCH В§"Pattern 4"):

1. `vault.on('modify', file)` fires.
2. Plugin handler queries `widgetRegistry.getByFilePath(file.path)` вҖ” if no match, no-op.
3. Reads fresh disk content; computes fence body hash via `sha1(extractFenceBody(disk, fenceIndex))`.
4. **Pre-suppression early-return (Pitfall P2):** if `observedFenceHash === widget.currentDocHash`, no widget action (frontmatter-only writes hit this path).
5. `selfWriteSuppression.tryConsume(path, observedHash)`:
   - `'consumed'` вҶ’ no widget action (this widget's own debounced write).
   - `'stale' | 'miss'` вҶ’ check `widget.hasUnflushedEdits()`:
     - `false` вҶ’ `widget.reloadFromDisk('silent')` (D-conflict-03 line/col clamp cursor preservation).
     - `true` вҶ’ `new ConflictModal(app, widget, mineDoc, externalDisk).open()` (SYNC-05, D-conflict-01).
6. Multi-widget case (multi-pane): iterate all widgets for `file.path`.

User-visible feedback (silent reload): cursor stays on same line:col (clamped to new doc bounds); scroll preserved; no Notice.
