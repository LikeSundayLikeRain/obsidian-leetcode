## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External file write → Widget reload | Untrusted disk content (could be from Obsidian Sync, CLI tools, malicious user) crosses into the widget's EditorView via `reloadFromDisk` and into the ConflictModal's diff render |
| Widget → Disk ("Keep mine") | "Keep mine" path overwrites disk content with widget doc — could destroy external user's edits |
| ConflictModal DOM render | The External pane renders untrusted disk content as `textContent` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03-01 | Information Disclosure (XSS) | ConflictModal renders external file content | mitigate | All text rendering via `textContent` / `createEl(..., { text })` — NEVER `innerHTML`. CLAUDE.md no-innerHTML lint rule (`eslint-plugin-obsidianmd`) enforces. Diff column rendering uses `<span class="lc-diff-{kind}">` with `textContent` only. Per Modal Test 7 + Step 1 verification (grep file for `innerHTML` returns zero hits). |
| T-20-03-02 | Tampering | "Keep mine" overwrites external user edits without confirmation | accept | The diff view IS the confirmation surface (D-conflict-02 + UI-SPEC §"Destructive actions" lines 142-150). | T-20-03-04 | Denial of Service | Pathological LCS input (huge files, all-different lines) → O(m·n) DP table | mitigate | Input is bounded by fence body size (~150 lines typical; 1000-line worst case).
