<objective>
Tackle the post-UAT blocker that all four prior gap-closure plans (20-05/06/07/08) failed to
resolve: typing in the widget loses focus after the 400ms flush. Runtime tracing under UAT
confirmed the root cause is structural to v1.3's "child types → debouncedWriter →
vault.process → Obsidian reloads parent CM6 from disk" pipeline. Even when the ViewPlugin's
suppression-map gate fires correctly (verified `match: true` in console), the parent CM6
docChange transaction transiently detaches the widget's contentDOM during line reflow, which
forces blur. Refocus on `requestAnimationFrame` is too late (CM6's measure phase has already
fired the blur).

The fix mirrors the Tasks plugin pattern (researched community plugins): the widget's
typed input dispatches as a range-remapped change directly onto the parent CM6 doc via
EditorView.dispatch with a `userEvent: 'leetcode.child-sync'` annotation. The parent doc
stays in sync with the child in real time. Disk persistence is handled by Obsidian's
standard editor auto-save (~250ms) on the parent — we never call vault.process from the
typing path, so there is no disk-driven parent reload to detach the widget DOM.

This is an architectural amendment to 20-CONTEXT.md L3 (which framed Compartment.reconfigure
as the live-config swap) and to the v1.3 milestone tagline ("file is the single source of
truth"). Refined statement: file remains the persistent single source of truth for external
readers; the parent CM6 is the authoritative in-memory editing surface for the typing path.
The child editor inside Decoration.replace is a remapped view onto the parent's fence-body
range. Bidirectional sync (child→parent on every keystroke, parent→child on external writes)
echo-prevented via annotations — same pattern v1.2 used in production across Phases 13-17,
modulo the multi-tab dual-CM6 complexity which 20-CONTEXT.md L10 single-active-per-file
already simplifies away.

Bug B body-swap on language switch is folded into this plan because copyToCode wrote to the
wrong fence (it found the first ```lang``` block under ## Code, not the leetcode-solve
block). The new path: switchLanguageFromWidget dispatches onto the parent CM6 at the
leetcode-solve fence body range with the LC API starter code, plus a frontmatter write.
The parent dispatch flows through the existing parent→child sync path (with syncAnnotation)
so the child editor reflects the new starter code without remount.

Purpose: focus survives every flush in single-pane and multi-pane; language switch writes
the right fence; conflict modal still triggers on external writes.
Output: net code reduction (~150 LOC) — most of selfWriteSuppression's peek path, the
ViewPlugin gate, and the eq() suppression clause retire. New: ~100 LOC childParentSync.ts.
</objective>
