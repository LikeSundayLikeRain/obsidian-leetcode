<objective>
Fix Cluster A (registry-key collision + Hook 1 echo amplifier).

Two compounding defects produce the multi-pane Take-Over CTA asymmetry (gap 3) and the
widget-thrash-on-type (gap 4) blockers reported in `20-HUMAN-UAT.md` Test 4.

  PRIMARY: `src/widget/WidgetController.ts:922` registers controllers under
  `${file.path}::${fenceIndex}` — pane-blind. When the same LeetCode note is open in two
  panes, both widgets compute the same key; `Map.set` clobbers; only one survives in
  `widgetRegistry.values()`. The multi-pane coordinator sees only the surviving entry → CTA
  asymmetry. The destroy(_dom) path looks up the same lossy key → can delete the OTHER
  pane's entry → mount/unmount thrash.

  AMPLIFIER: `src/main.ts:962-966` Plan 19-02 Hook 1 calls `widgetRegistry?.flushAll()`
  synchronously on EVERY active-leaf-change — including focus reaffirmations within the same
  leaf produced by mousedown→focus inside the widget. flushAll → vault.process echo → parent
  CM6 ViewPlugin rebuild (`liveModeViewPlugin.ts:110-116`) → eq() false → CM6 destroy+remount
  on every keystroke.

This plan fixes BOTH together because they share a root and any partial fix leaves the other
symptom visible. The key shape changes from `${file.path}::${fenceIndex}` to
`${file.path}::${fenceIndex}::${leafId}` where `leafId` is a stable identifier resolved at
mount time. Hook 1 is gated on actual file-path transitions.

Purpose: Phase 20 Test 4 (blocker UAT) passes — symmetric CTA + stable typing in both panes.
Output: Mounted controllers per-pane unique in registry; Hook 1 stops echoing on focus
reaffirmation; new regression test pins the production mount path.
