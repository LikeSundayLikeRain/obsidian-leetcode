# Plan 05.1-03 — D-10 Live-Smoke Checkpoint

## Outcome

**Resume signal:** `approved with spacing tweak` (plus three follow-on fixes surfaced during live smoke).

All 12 checklist items verified PASS in the developer's Leetcode vault. Three cosmetic / regression issues surfaced and were fixed in the same live-smoke cycle:

1. **Widget placement** — original anchor at closer-line end (`side: 1`) rendered visually clipped inside the rendered CM6 code-block widget. Fixed by anchoring at next-line start (`side: -1`) so the button row renders on its own line below the fenced block (matches reading-mode `insertAdjacentElement('afterend', ...)` semantics).
2. **Edit-Mode spacing** — Edit Mode had zero visual gap between fence and buttons because the CM6 code-block widget has no bottom margin. Added `.cm-editor .leetcode-code-actions { margin-top: 12px }` scoped override + compensating `-4px` in `.markdown-rendered pre + .leetcode-code-actions` so Read Mode's larger gap (driven by `<pre>`'s built-in `margin-bottom`) matches the new 12px Edit-Mode target.
3. **Problem Browser regression** — pre-existing but surfaced during live smoke: `activeDocument.createSvg('svg')` threw `"Only one element on document allowed"` because Obsidian's helper auto-appends to the document root (which already has `<html>`). Fixed by switching to `doc.createElementNS(SVG_NS, ...)` which creates detached elements we append ourselves. Not a Phase 5.1 regression — but since live smoke caught it, resolved here rather than filing a follow-up.
