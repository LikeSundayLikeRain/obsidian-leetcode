<objective>
Promote backlog 999.4 (relative line numbers in child editor) into the v1.2 ship gate per CONTEXT D-30. Add a plugin-owned `showRelativeLineNumbers: boolean` setting (default `false` per D-35) with a Settings tab toggle under the existing 'Code editor' section. When enabled (AND Obsidian's `showLineNumber` is also enabled), the child editor's gutter renders relative line numbers via `lineNumbers({ formatNumber: relativeFormatter })`: the cursor's line shows its absolute number, and other lines show their absolute distance from the cursor — vim `relativenumber` convention.

Per D-35, the setting is plugin-owned (NOT a wrapper around any third-party plugin's setting). Read-once-at-mount semantic per D-18 / Plan 17-12 — toggling the setting requires note remount (Cmd-E flip in/out of Source/Live Preview, OR close+reopen). The existing `lineNumbersEnabled` gate from Plan 17-12 LINENUM-01 is preserved verbatim — relative numbers layer ON TOP of it; absolute numbers stay the default behavior when the new setting is OFF.

Vim interaction (per CONTEXT.md §18-03): when both vim mode + showLineNumber + showRelativeLineNumbers are ON, the gutter renders relative numbers AND vim's runtime `:set nu` / `:set nonu` / `:set rnu` / `:set nornu` continue to work via `@replit/codemirror-vim`'s package handlers. Plan 18-01's `:set nu` / `:set nonu` aliases (registered via Vim.defineEx in childEditorVimScope.ts) interoperate naturally because the alias dispatches to the same package handler.

Purpose: vim users who rely on relative line numbers for jump-to-line operations (`5j`, `12k` based on the visible distance) currently see the third-party Obsidian relative-line-numbers plugin NOT propagate to the child editor (verified live during Phase 17 UAT Test 24 stretch — the third-party plugin targets parent CM6 only). Shipping a plugin-owned setting closes that gap inside our own plugin's surface.

Output: extended SettingsStore with a new boolean field + accessors; extended SettingsTab with a new Setting block; modified childEditorFactory.ts to pass `formatNumber: relativeFormatter` when the new gate is true; new pure `relativeFormatter` function (inline or sibling module); extended childEditorFactory.test.ts with conditional + formatter unit tests.
