<objective>
Add a plugin-owned settings toggle "Show relative line numbers in code editor" (default OFF) that switches the child editor's gutter from absolute line numbers to relative-to-cursor numbers via CM6's `lineNumbers({ formatNumber })` API. The fix layers on top of Plan 17-12's existing `showLineNumber`-gating contract: when Obsidian's global `showLineNumber` is OFF, no gutter renders at all (regardless of relative); when ON, the new plugin-owned toggle decides between absolute and relative formats. Read-once-at-mount semantic per D-35 (matches the existing D-18 vim-mode and showLineNumber contracts).

The previous Phase 18 attempt (commit 60c5cdf, reverted in dc886a1) failed because it added the factory parameter but missed the integration touchpoint at NestedEditorWidget.toDOM. This plan's acceptance criteria explicitly verify ALL FIVE integration touchpoints are wired (per 18-RESEARCH §4.3).

Purpose: Closes UAT Test 25 (LINENUM-RELATIVE-01) for v1.2 ship gate. Provides relative-line-number rendering inside the child editor independently of any third-party Obsidian plugin (D-35).
