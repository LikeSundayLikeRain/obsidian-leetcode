expected: Under "AWS Bedrock" provider settings, switch Auth method to "Explicit access keys" and enter fake values for Access key ID and Secret access key. Switch Auth method to "Default credential chain" — fake values disappear from the visible UI but stay in `data.json`. Switch back to "Explicit access keys" — original fake values STILL appear. Pitfall 10 invariant holds (mode switch only changes which fields render, never which fields are stored).
result: pass
note: User confirms behavior matches spec but flags the persist-on-switch as undesired UX. Captured as v1.2 polish follow-up — changing it now would contradict the locked CONTEXT.md decision B + Pitfall 10 mitigation (designed to prevent silent secret loss on accidental mode toggles).
