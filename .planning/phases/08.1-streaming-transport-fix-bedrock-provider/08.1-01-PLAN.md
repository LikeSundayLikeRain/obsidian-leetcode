Restore real token-by-token streaming to AI Debug by adding a NEW PRIMARY transport tier — native `window.fetch()` — at the top of `AIClient.invokeStream`'s decision tree. The current Phase 08 stream path (TIER 2 = `obsidianFetch('stream')` → `electron.net.fetch`) ALWAYS throws on Obsidian 1.10+ because `electron.net.fetch` is a main-process API blocked by `contextIsolation: true`, so every AI Debug call falls through to the buffered "Thinking…" tertiary. The new TIER 1 makes `AIClient.invokeStream`'s `consumeStream` branch live for the first time. Plan also installs a sibling LC-isolation gate (grep + fs-walk runtime test) so the new "native fetch is legal in src/ai/" surface cannot leak into LC paths and silently break AIPROV-05.

Purpose: Phase 08's eager-probe in `obsidianFetch.ts:loadElectronNet()` correctly throws on contextIsolation:true (kept as future-proofing seam); but the primary transport must move ABOVE that probe so live streaming works in real Obsidian. Plan 08.1-02 inherits this transport for free.

Output: Modified `AIClient.invokeStream` with TIER 1 native-fetch insertion; new shell grep gate; 3 new unit test files exercising the decision tree; 1 new fs-walk regression test.
