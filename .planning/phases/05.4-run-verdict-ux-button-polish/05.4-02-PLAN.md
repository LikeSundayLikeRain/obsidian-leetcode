D-01 join formula (CONTEXT.md):
> "RunModal joins all tabs' inputs into one `dataInput` string"

D-02 arity formula (CONTEXT.md + RESEARCH.md §Q3):
> "Per-case arity is derived from sampleTestCase (newlines + 1 = lines per case). metaData.params.length is the canonical K source; sampleTestCase newlines+1 is the fallback when metaData is absent."

For the JOIN side specifically, the LC wire is just a flat newline-joined string of all per-case lines. Since each `this.cases[i]` IS already the per-case lines, the simplest correct join is `cases.map(c => c.trim()).filter(c => c.length > 0).join('\n')` — arity is not strictly required for the JOIN direction. Arity matters for the SPLIT direction (Plan 03). The plan therefore uses `joinCasesForRun(this.cases, arity_or_cases_length_fallback)` defensively.
