<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| plugin → child_process | Untrusted command from user's ~/.aws/config spawned via spawnSync |
| child_process stdout → plugin memory | Raw credentials flow through; must never reach logger |
| child_process stderr → Notice UI | Potentially secret-bearing diagnostic output; truncated to 200 chars |
| process.env → plugin | AWS env vars read at runtime; plugin never WRITES env vars |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.2-01 | Information Disclosure | credentialProcess.ts stdout | mitigate | D-22: stdout NEVER passed to logger; only flows through JSON.parse -> shape validator -> cache. Test asserts no logger call contains stdout content. |
| T-08.2-02 | Information Disclosure | credentialProcess.ts stderr | mitigate | D-08: stderr truncated to 200 chars at source boundary in runCredentialProcess. Downstream receives already-truncated string. |
| T-08.2-03 | Tampering | credential_process command injection | mitigate | D-20: shell:false prevents shell metachar interpretation. D-21: parseCommandLine handles double-quoted args only — no single-quote, no $(), no &&. Command comes from user's own ~/.aws/config (user-controlled, not attacker-controlled in normal threat model). |
| T-08.2-04 | Denial of Service | credential_process hung process | mitigate | D-20: timeout:30_000 kills child after 30s. Error message: 'credential_process timed out after 30s'. Single concurrent spawn per profile via D-12 coalescing. |
| T-08.2-05 | Information Disclosure | cache entry leakage across profiles | accept | Cache keyed by profile name (D-14). Switching profiles returns only that profile's cached creds. Single-user desktop app — no cross-user threat. |
| T-08.2-SC | Tampering | npm/pip/cargo installs | accept | Phase 08.2 installs ZERO new packages. No registry interaction. |
</threat_model>
