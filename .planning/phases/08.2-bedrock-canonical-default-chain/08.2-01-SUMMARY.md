## Implementation Details

### awsCredentials.ts Rewrite (Task 1)

- **readEnvCredentials()**: Returns env creds or null
- **resolveProfileName()**: AWS_PROFILE -> AWS_DEFAULT_PROFILE -> "default" (D-17 step 2)
- **resolveCredentialsPath()**: Honors AWS_SHARED_CREDENTIALS_FILE
- **resolveConfigPath()**: Honors AWS_CONFIG_FILE
- **safeReadFile()**: fs.readFileSync wrapped in try/catch, returns '' on failure (D-18)
- **parseIniSection()**: Dispatches by fileType — credentials uses bare `[name]`, config uses `[profile name]` with `[default]` exception
- **mergeProfileSettings()**: Config wins region, credentials wins keys, credential_process from either
- **resolveAwsCredentials()**: Unchanged public signature; full waterfall internally

### credentialProcess.ts (Task 2)

- **parseCommandLine()**: Double-quote parser with backslash-escape; empty input throws
- **runCredentialProcess()**: spawnSync with D-20 hardened options (shell:false, timeout:30_000, windowsHide:true)
- **parseCredentialProcessOutput()**: Strict AWS CLI v1 JSON shape validation (Version/AccessKeyId/SecretAccessKey/SessionToken/Expiration)
- **getCachedOrRefreshSync()**: Synchronous cache lookup + spawn with discriminated-union cache (fresh/refreshing), 5-min SKEW_MS, 24h default TTL
- **getCachedOrRefresh()**: Async wrapper with Promise-based concurrent coalescing (D-12)
- **clearCredentialProcessCache()**: Test teardown utility
