## Q&A

### Q1 — Plan split
**Options:** 5 plans by concern / 3 plans by phase / single mega-plan
**Selection:** 3 plans by phase
**Notes:** Phase 22 is "deletion + checklist" per ROADMAP §22 LOW research flag. 5 plans was finer granularity but 3 of those collapse to checklists. Single mega-plan loses `gsd-undo` granularity. 3 plans (22-01 cutover; 22-02 polish; 22-03 release gates + BRAT) is the right balance. Captured as D-plan-01.

### Q2 — Cutover order
**Options:** Flip default first / Delete first / Single atomic
**Selection:** Flip default first, delete after
**Notes:** Safety-net argument is decisive — Phase 21.1 R6 was exactly the class of bug that surfaces only against `=ON`-default-fresh-install behavior. Flip-first lets us discover regressions WITH the v1.2 path still alive (toggle OFF, fix, retry). After deletion, recovery is `git revert` only. Captured as D-cutover-01 with internal sub-step staging.

### Q3 — Vim-Tab fix scope
**Options:** Investigate + fix in Phase 22 / Defer entirely / Hard requirement
**Selection:** Investigate + fix in Phase 22 (with v1.3.x defer fallback)
**Notes:** ROADMAP §22 success criterion 7 says "cosmetic only" — not a release blocker. Hard requirement risks Phase 22 stall on undocumented CM5 adapter. Defer-entirely punts a known-bad surface to GA. 30-min time-boxed probe of `Vim.handleKey('<Tab>', getCM(view))` on the existing CM5 adapter; ship if it works, defer to v1.3.x backlog if not. Captured as D-polish-01.

### Q4 — THEME-05 verification
**Options:** Manual checklist / Automated visual-diff harness / Spot-check 2 themes
**Selection:** Manual visual checklist
**Notes:** Automated harness is v1.4+ infra investment (Obsidian-headless visual-diff is unsolved). Spot-check is too narrow — ROADMAP names 5 themes specifically. Manual checklist matches "side-by-side" framing; v1.2 baseline regenerated by `git checkout` of v1.2 ship commit. Captured as D-gate-03.

### Q5 — VIM-03 banner
**Options:** Skip entirely / Ship as defensive fallback / Ship unconditionally
**Selection:** Skip VIM-03 entirely
**Notes:** Phase 20 live-reconfigure works (per 20-CONTEXT). VIM-03 is conditional in ROADMAP §22 ("only if Phase 20 proved unreliable"). Mark VIM-03 as "Resolved by Phase 20 live-reconfigure" in REQUIREMENTS.md traceability. Captured as L7.

### Q6 — BRAT alpha duration
**Options:** 1-week dogfood / 2-week with explicit feedback / Skip BRAT
**Selection:** 1-week dogfood + watch GitHub Issues
**Notes:** Single-author plugin discipline matches v1.0/v1.1/v1.2 ships. 2-week + feedback solicitation is ship delay without proportional signal. Skip-BRAT loses real-vault smoke for an architectural change. Captured as L5 / D-gate-04.

### Q7 — Bundle gate
**Options:** Hard cap at 1.71 MB raw / Soft warning / Raw + gzipped both
**Selection:** Hard cap at 1.71 MB raw
**Notes:** POLISH-02 "measurably smaller" — hard cap operationalizes. Soft warning loses the guarantee. Both raw + gzipped is marginal upside. Phase 22 expected to land well under (~1.5 MB target given net −2,400 LOC). Captured as L3 / D-gate-01.

### Q8 — `useNestedEditor` settings cleanup
**Options:** Read-and-ignore / Active migration / Deprecated alias
**Selection:** Read-and-ignore
**Notes:** Simplest correct approach; zero risk; matches "user owns their data." Active migration adds testing burden for a one-shot cleanup. Deprecated alias carries v1.2 vocabulary forward. Persisted field disappears on next save naturally. Captured as L6 / D-settings-01.

### Q9 — Migration infrastructure post-cutover
**Options:** Keep indefinitely / Delete in v1.4+ sunset / Delete in Phase 22
**Selection:** Keep all of it indefinitely
**Notes:** Users who skip 1.3.0 and upgrade later from 1.2.x still need lazy migration. Phase 21 CONTEXT §specifics confirms "migrator + backup GC + banner stay." Deletion in Phase 22 strands users who delay upgrades. Captured as L2 / D-settings-02.

### Q10 — `'leetcode.*'` userEvent annotations
**Options:** Strip them all / Keep for diagnostics / Strip only dead
**Selection:** Strip them all
**Notes:** PROTECT-03 retires the convention; dead annotations on live dispatches contradicts that. v1.3 `sectionProtectionExtension` doesn't filter on userEvent — annotations are no-ops. Cost: few hundred bytes. Benefit: codebase that means what it says. Captured as D-unwire-02.

### Q11 — main.ts unwiring strategy
**Options:** Single atomic commit / Multiple staged / Codemod
**Selection:** Single atomic commit
**Notes:** Half-unwired main.ts doesn't compile (every `useInlineWidget` branch references either v1.2 or v1.3 code). Multiple staged commits churn intermediate states that break tsc. Codemod is over-engineering for a one-time strip. Manual + grep + tsc fast-feedback is sufficient. Captured as D-unwire-01.
