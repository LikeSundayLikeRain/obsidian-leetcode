    Create src/contest/ContestFinalizer.ts:

    Export interface FinalizeContestArgs:
    - session: ContestSession (snapshot from abort/finish)
    - aborted: boolean (true if user aborted)
    - app: App
    - settings: SettingsStore
    - noteWriter: NoteWriter

    Export async function finalizeContest(args: FinalizeContestArgs): Promise<string> (returns summary note path):

    1. Compute actual solving duration: totalElapsedMs = session.isPaused ? (session.pausedAt! - session.startedAt - session.pausedDuration) : (Date.now() - session.startedAt - session.pausedDuration). Cap at duration*1000.

    2. Determine problems folder: settings.getProblemsFolder(). Contest subfolder: `{folder}/Contests/{session.contestSlug}/`

    3. For each problem in session.problems where code !== '':
       a. Get cached detail: settings.getProblemDetail(problem.slug)
       b. If detail is null: skip (shouldn't happen since we fetched on start, but defensive)
       c. Build canonical path: buildNotePath(contestSubfolder, detail.id, problem.slug)
       d. Check if existing file at that path OR at the normal problems folder path
       e. Apply D-13 merge strategy:
          - If existing file AND verdict === 'accepted': overwrite ## Code section via app.vault.process(file, body => rewriteCodeSection(body, problem.code, problem.language))
          - If existing file AND verdict !== 'accepted': DO NOT touch the existing note (failed attempt captured in summary only)
          - If no existing file: create new note via app.vault.create at contestSubfolder path, build body with buildNoteBody, apply frontmatter with lc-contest-id
       f. Apply lc-contest-id frontmatter to new/overwritten notes: app.fileManager.processFrontMatter(file, fm => { fm['lc-contest-id'] = session.contestSlug; })

    4. Tag missed problems with #revisit (CONTEST-08):
       For each problem where verdict !== 'accepted':
       - Find the file (either just-created or existing)
       - app.fileManager.processFrontMatter(file, fm => { const tags = Array.isArray(fm.tags) ? fm.tags : []; if (!tags.includes('revisit')) { tags.push('revisit'); fm.tags = tags; } })

    5. Write summary note:
       - Path: `{folder}/Contests/{date}-{session.contestSlug}.md` where date = YYYY-MM-DD from session.startedAt
       - Ensure Contests/ folder exists (app.vault.createFolder if needed — handle EEXIST)
       - Build body using buildSummaryBody helper:
         - H1: session.contestTitle
         - Aborted marker if args.aborted: "**(aborted at {remainingMM}:{remainingSS} remaining)**"
         - Score line: "**Score:** {scored}/{total} ({solvedCount}/4 solved)" where scored = sum of credit for accepted problems, total = sum of all credits
         - Duration line: "**Duration:** {M}m {S}s"
         - "## Results" heading + Markdown table: Problem | Difficulty | Verdict | Time | Points
         - Per-problem row: [[slug]] | Easy/Medium/Hard | Accepted/Attempted/Unsolved | time-to-solve or '—' | credit or '0'
         - "## Notes" heading (empty — user fills)
       - Create file: app.vault.create(summaryPath, body)
       - Apply summary frontmatter: app.fileManager.processFrontMatter(summaryFile, fm => { fm['lc-contest-id'] = session.contestSlug; fm['lc-contest-type'] = session.contestType; fm.date = dateStr; fm.duration = Math.round(totalElapsedMs/1000); fm.score = scored; fm['solved-count'] = solvedCount; fm.problems = session.problems.map(p => p.slug); })

    6. Return summary note path (for Notice in Plan 05 callers)

    Export helper function buildSummaryBody(args: { session, aborted, totalElapsedMs }): string — pure function for testability

    Export helper function rewriteCodeSection(body: string, code: string, language: string): string — finds ## Code heading, replaces the first fenced code block after it with new code (using same codeBlockFor pattern from NoteTemplate)
