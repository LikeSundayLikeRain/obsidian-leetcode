# Phase 10 — UI Design Contract

> Visual and interaction contract for the Contest (virtual + analysis) phase. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

This is an Obsidian community plugin running inside the Obsidian desktop app. The plugin owns chrome (custom `ItemView`, settings tab, modals) and inherits all chrome-adjacent surfaces from Obsidian (tab strip, scroll container, focus rings, theme palette). Phase 10 adds a virtual contest mode with timer, ephemeral editing, and AI analysis — entirely within the existing plugin architecture.

**Scope covered by this contract:**
1. Modified `ProblemBrowserView` — contests/problems mode toggle at top of sidebar
2. Contest list rendering — searchable rows within the contests mode
3. Contest preview — details + "Start Contest" before committing (reuses preview pattern)
4. Contest active state — sticky timer header with countdown + per-problem verdict badges
5. `ContestSolveView` — dedicated `ItemView` for solving (code editor + Run/Submit buttons)
6. Abort confirmation modal — destructive action: abort contest
7. Summary note layout — rendered Markdown with `## Results` table + `## AI Analysis` + `## Notes`
8. Settings toggle — `Auto AI contest analysis` boolean
9. Palette commands — `start-random-contest`, `generate-contest-analysis`, `pause-contest`, `abort-contest`
10. `Notice` toasts — contest lifecycle events
11. AI analysis streaming — reuse `AIStreamModal` for manual re-run

**Explicitly out of scope** (deferred):
- Live contest participation (CONTEST-FUT-01)
- Difficulty-weighted Surprise me (CONTEST-FUT-02)
- Upcoming contest schedule (CONTEST-FUT-03)
- Per-feature AI provider routing (AIPROV-FUT-02)

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian-native) |
| Preset | not applicable — Obsidian plugin, no React/Next/Vite stack |
| Component library | Obsidian API primitives (`ItemView`, `Modal`, `Setting`, `Notice`, `Menu`, `setIcon`, `MarkdownRenderer`) |
| Styling approach | Class-scoped plain CSS in `styles.css` rooted at `.leetcode-contest` (contest surfaces) and extending `.leetcode-browser` (mode toggle). Reuses shared `lc-diff--*` selectors and `.lc-empty` pattern. |
| Token system | Obsidian CSS variables (`--background-primary`, `--background-secondary`, `--text-normal`, `--text-muted`, `--interactive-accent`, `--color-green`, `--color-yellow`, `--color-red`, `--background-modifier-border`, `--text-on-accent`, `--text-error`, `--text-success`). **Never hardcoded hex.** |
| Icon library | Lucide (Obsidian built-in) — `setIcon(el, name)`. Phase 10 uses: `trophy` (contest tab icon), `timer` (countdown display), `play` (start/resume), `pause` (pause), `square` (abort/stop), `shuffle` (surprise me, reused from existing pattern), `check-circle` (AC badge), `x-circle` (failed badge), `circle` (unsolved badge), `sparkles` (AI analysis) |
| Font | Inherit Obsidian's `--font-interface` and `--font-monospace`. No custom web fonts. |
| DOM constructor | `createEl()` / `createDiv()` / `createSpan()` only. `innerHTML` forbidden (`eslint-plugin-obsidianmd@^0.3.0` enforced). |

## Spacing Scale

Obsidian-aligned 4-point scale. Reuses tokens already established in prior phases. Phase 10 does NOT introduce new spacing tokens.

| Token | Value | Usage in Phase 10 |
|-------|-------|-------------------|
| xs | 4px | Gap between verdict badges in timer header, gap between contest row elements |
| sm | 8px | Gap between mode toggle buttons, internal padding of timer display, gap between problem cards in solve view |
| md | 12px | Gap between timer sections (clock + badges), topbar margin-bottom, contest row padding |
| md2 | 16px | Outer padding of contest view containers, sticky header padding, modal body padding |
| lg | 24px | Body content top padding below sticky header, section breaks in summary note |
| xl | 32px | Major visual separation between timer header and problem list |

Exceptions:
- **Mode toggle button height: 28px** — slightly smaller than icon buttons (32px) to read as a tab-toggle rather than an action button.
- **Timer countdown font: 20px** — larger than body (14px) for at-a-glance readability during a timed contest.
- **Verdict badge: 20px diameter** — matches existing `.leetcode-submissions-chip` height (20px) from `styles.css:802`.

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 10 |
|------|------|--------|-------------|-------------------|
| Body | 14px (`var(--font-ui-small)`) | 400 | 1.5 | Contest row descriptions, settings descriptions, modal body text |
| Label | 12px | 400 | 1.4 | Contest row metadata (date, type), verdict badge labels, timer "remaining" label |
| Heading | 18px | 600 | 1.3 | Contest title in preview/active header, summary note file title in palette |
| Timer | 20px (`var(--font-monospace)`) | 600 | 1.2 | Countdown `MM:SS` display — monospace for stable digit width |

**Two weights only:** 400 (regular) for body and labels + 600 (semibold) for headings and timer. Labels at 12px/400 read as subordinate metadata without needing a distinct weight — their smaller size provides sufficient visual differentiation from 14px body text.

**Monospace for timer:** The countdown display uses `font-variant-numeric: tabular-nums` combined with `var(--font-monospace)` to prevent jitter as digits change (same pattern as `.lc-row__acrate` at `styles.css:175`).

**Hard rule (carried from v1.0):** every color value in Phase 10 CSS references an Obsidian CSS variable. **No raw hex. No `rgba()` with literal numbers.** Non-negotiable.

**Accent (`var(--interactive-accent)`) reserved EXCLUSIVELY in Phase 10 for:**
1. "Start Contest" button in contest preview (primary CTA)
2. Active mode toggle segment indicator (border-bottom or background)
3. Timer progress bar fill

**NOT accent in Phase 10:**
- Pause/Resume/Finish buttons (neutral — routine actions during contest)
- Abort button (destructive — uses `--text-error` color)
- Contest row hover states (uses `--background-modifier-hover`)
- Verdict badges (use semantic green/red/faint)

**60/30/10 mapping:**
- 60% `--background-primary`: all view backgrounds, timer header, modal chrome
- 30% `--background-secondary`: contest rows, code editor, inactive toggle segment, problem cards
- 10% `--interactive-accent`: Start Contest button, active toggle indicator, timer bar

## Interaction Contracts

### Mode Toggle (ProblemBrowserView modification)

| Interaction | Contract |
|-------------|----------|
| Click "Contests" segment | View switches to contest mode: contest list renders, search resets, shuffle button picks contests. `mode` stored in memory (not PluginData). |
| Click "Problems" segment | View returns to problem mode (existing behavior preserved byte-for-byte). |
| Active contest exists on view open | Auto-switch to contests mode; render active contest timer + problems instead of contest list. |
| Plugin reload with active contest | `onOpen()` checks `settings.getContestSession()`. If non-null and active, auto-switch to contests mode. |

### Contest List (within Contests mode)

| Interaction | Contract |
|-------------|----------|
| Search input | Filters contest list by title (case-insensitive substring match). Debounce 200ms. |
| Click contest row | Opens contest preview (reuse `openOrReusePreview` pattern with contest state, OR a modal — planner decides). |
| Shuffle button (Surprise me) | Picks random contest from cache; validates 4 problems fetchable (max 3 retries); on success opens preview; on failure shows Notice. |
| Scroll to bottom | Loads next page of contests from cache (or fetches next page if cache exhausted). Pagination: 50 per page. |
| Pull to refresh / manual refresh | Refresh contest index from API, reset 24h TTL. |

### Contest Preview (before starting)

| Interaction | Contract |
|-------------|----------|
| Click "Start Contest" | Disable button (label: "Starting…"). Fetch all 4 problem details. On success: create `ContestSession` in PluginData, open active contest view, close preview. On failure: re-enable button, Notice. |
| Click outside / close | Preview closes. No state change. |
| Problem rows in preview | Non-interactive (display only — shows title + difficulty). |

### Active Contest (timer header + problem navigation)

| Interaction | Contract |
|-------------|----------|
| Timer tick | Re-render `MM:SS` every 1000ms via `registerInterval`. Color shifts: normal (> 10 min), `--color-yellow` (5-10 min), `--color-red` (< 5 min). |
| Click problem badge | Opens that problem in `ContestSolveView` (or switches to it if already open). |
| Click "Pause" | `isPaused = true; pausedAt = Date.now()`. Timer stops. Label changes to "Resume". Timer text shows "Paused" in `--text-muted`. |
| Click "Resume" | `pausedDuration += Date.now() - pausedAt; isPaused = false; pausedAt = null`. Timer resumes. Label changes to "Pause". |
| Click "Finish" | Finalize contest: write problem notes, write summary note, trigger AI analysis if toggle ON. Show completion Notice. Return to contest list. |
| Click "Abort" | Open abort confirmation modal. |
| Timer expires (remaining = 0) | Auto-finalize: same as Finish. Timer shows "00:00". Notice: "Time's up!" |
| Plugin reload | Timer header re-renders from persisted `ContestSession`. Remaining computed from wall clock. |

### ContestSolveView (editing surface)

| Interaction | Contract |
|-------------|----------|
| Navigate to problem | View loads problem content from cached details + code from `ContestProblemState.code`. |
| Edit code | Updates `ContestProblemState.code` in memory. Debounced save to PluginData every 30s. |
| Click "Run" | Uses existing `leetcodeRest.interpretSolution` with slug, code, language from contest state. Opens VerdictModal (existing). |
| Click "Submit" | Uses existing `leetcodeRest.submit` pipeline. On AC: `verdict = 'accepted'; solvedAt = Date.now()`. Badge in timer header updates. |
| Language selector | Chevron dropdown (reuses existing `.leetcode-language-chevron` pattern). Persists to `ContestProblemState.language`. |
| Tab close | View closes but contest state persists in PluginData. User can re-open by clicking the problem badge. |

### Abort Modal

| Interaction | Contract |
|-------------|----------|
| Click "Abort contest" (confirm) | Finalize with aborted flag. Write summary with "(aborted)" marker. Close modal. Return to contest list. |
| Click "Cancel" | Close modal. Contest continues. |
| Press Escape | Same as Cancel. |
| Click overlay | Same as Cancel. |

### Settings (new toggle)

| Interaction | Contract |
|-------------|----------|
| Toggle on | `autoAIContestAnalysis = true`. Persists immediately to PluginData. |
| Toggle off | `autoAIContestAnalysis = false`. Persists immediately. |
| Position | After `AI` section heading block, before any future section. New `Contest` sub-heading. |

## Accessibility

| Requirement | Contract |
|-------------|----------|
| Keyboard navigation | Mode toggle buttons are `<button>` (focusable). Timer action buttons are `<button>`. Problem cards are `<div role="button" tabindex="0">` with Enter/Space handlers. |
| Focus rings | Do NOT override Obsidian's default focus styling. All buttons inherit theme focus ring. |
| ARIA on mode toggle | `role="tablist"` on container, `role="tab"` + `aria-selected` on each button. |
| ARIA on timer | `aria-live="polite"` on timer display for screen reader countdown updates (debounced to every 60s to avoid noise). `aria-label="Contest timer: {MM} minutes {SS} seconds remaining"`. |
| ARIA on verdict badges | `aria-label="{problem-title}: {verdict}"` on each badge. |
| ARIA on abort modal | Inherits from Obsidian `Modal` — focus trapped, Escape closes. |
| Color contrast | All colors via Obsidian theme variables — themes meet WCAG AA. Timer warning/critical colors supplement (not replace) the digit rendering. |
| Motion | Timer tick is text-only update (no animation). Progress bar uses `transition: width 1s linear` — gated by `@media (prefers-reduced-motion: no-preference)` (same as `lc-spin` precedent). |
| Reduced motion | Timer progress bar transitions disabled under `prefers-reduced-motion: reduce`. |
| Screen reader for active contest | Status announcement on contest start/pause/resume/end via `aria-live` region. |

## State Machine — Contest Lifecycle

```
         ┌───────┐
         │ idle  │  (no contest active; PluginData.contestSession === null)
         └───┬───┘
             │ user clicks "Start Contest"
             ▼
     ┌────────────────┐
     │ starting       │  (fetching 4 problem details)
     └────┬──────┬────┘
     success│    │ failure
          ▼    ▼
  ┌──────────┐  ┌──────┐
  │ active   │  │ idle │ (Notice + re-enable button)
  └──┬───┬───┘  └──────┘
     │   │
  pause│ │timer expires / user clicks Finish / abort confirmed
     ▼   ▼
┌────────┐  ┌────────────┐
│ paused │  │ finalizing │  (writing notes + summary + AI)
└────┬───┘  └─────┬──────┘
     │ resume     │ complete
     ▼            ▼
┌────────┐  ┌───────┐
│ active │  │ idle  │  (session cleared from PluginData)
└────────┘  └───────┘
```

**Persistence rule:** `PluginData.contestSession` is saved on every state transition (start, pause, resume, verdict change, code edit debounce every 30s). On plugin reload, if state is `active` or `paused`, the view renders accordingly from persisted state.

## Edge Cases (locked decisions from CONTEXT.md)

| Case | Behavior |
|------|----------|
| User starts contest, closes Obsidian, reopens | Contest resumes from persisted state. Timer recomputes remaining from wall clock. |
| User switches to Problems mode during active contest | Allowed. Timer header only visible in Contests mode. Contest continues in background. Badge state updates on verdict. |
| User closes ContestSolveView tab | Code persists in PluginData. User re-opens by clicking problem badge. |
| Timer expires while user is submitting | Wait up to 10s for active submission to resolve before finalizing. |
| Surprise me picks a contest with unfetchable problems | Retry up to 3 times with different random contests. After 3 failures, Notice. |
| User already has notes for some contest problems | Per D-13: AC overwrites `## Code`; non-AC leaves existing note alone. |
| User clicks Finish with 0 problems attempted | Summary written with all "Unsolved". Score: 0/{total}. No `#revisit` tags needed (all are missed). |
| Active contest + user quits unexpectedly (crash) | PluginData persisted on last debounce (30s max data loss for code edits; timer state always fresh from epoch computation). |
| Two Obsidian windows (popout) | Contest state is global (PluginData). Timer header renders in whichever window has the contests tab open. Solve view opens in the window where the problem badge was clicked. |
