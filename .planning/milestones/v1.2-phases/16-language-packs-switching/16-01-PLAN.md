Builder contract (RESEARCH §1, §5):
```
export function buildLanguageExtensions(
  slug: string,
  override: 'auto' | 2 | 4 | 8,
): Extension[];

// Returns array shape:
//   [getLanguageSupport(slug),
//    indentUnit.of(effectiveIndent(slug, override)),
//    closeBrackets(),
//    keymap.of([{ key: 'Mod-/', run: toggleLineComment }])]

export function effectiveIndent(
  slug: string,
  override: 'auto' | 2 | 4 | 8,
): string;
// Go: '\t' always. Override !== 'auto': ' '.repeat(override) (except Go).
// 'auto': map per D-05.

export const languageCompartment: Compartment;
// Module-level singleton (Pitfall C — identity-keyed; safe across multiple child editors).
```

Slug → LanguageSupport call (from RESEARCH §1 table):
- python3 → python()
- java → java()
- cpp / c → cpp()
- javascript → javascript()
- typescript → javascript({ typescript: true })
- golang → StreamLanguage.define(go)
- rust → rust()
- default fallback (unknown slug, e.g. 'csharp' that won't be in the chevron) → python() with 4-space indent (defensive; D-04 says they shouldn't reach this path but the function should not throw)

Pitfall A (RESEARCH §12): Call `StreamLanguage.define(go)` INSIDE the function body, not at module top-level. This avoids invoking the external `@codemirror/language` before Obsidian provides it.

Pitfall B (RESEARCH §12): Import `closeBrackets` from `'@codemirror/autocomplete'` package root. NEVER from a deep subpath.

Pitfall C (RESEARCH §12): Module-level `languageCompartment` singleton is correct — Compartments are identity-keyed; effects target the EditorView the dispatch goes to.

Pitfall E (RESEARCH §12): If Wave 3 behavioral tests reveal that StreamLanguage(go) does not expose `commentTokens` for COMMENT-01, wrap with `new LanguageSupport(StreamLanguage.define(go), [languageData.of({ commentTokens: { line: '//' } })])`. Initial implementation does NOT pre-emptively add this — implement the simpler form first; if 16-05 behavioral test fails, add the wrap as a follow-up edit in this file.
