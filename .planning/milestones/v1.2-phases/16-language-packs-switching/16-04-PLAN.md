Extend `switchFenceLanguage` in `src/main.ts` (the chevron handler entry point at line ~2321) to dispatch a `Compartment.reconfigure` effect on the child EditorView whenever a child is registered for the current file. This is the load-bearing dispatch for LANG-01 (D-12).

Step B already uses `userEvent: 'leetcode.lang-switch'` (line 2392) — this plan adds a SECOND dispatch on the CHILD using the SAME userEvent (D-12 step 3).

Insertion point: BETWEEN Step B (parent dispatch line 2393) and Step C (processFrontMatter line 2401). Add a new "Step B′" comment block with the child reconfigure dispatch.

Insertion-point rationale (RESEARCH §5 + Pitfall 5): The parent dispatch is synchronous; the child dispatch is also synchronous; both run before yielding to processFrontMatter (which is `await`ed). If a developer later moves Step C above Step B′, the child reconfigure would still be correct (it's effects-only). Order between B and B′ does not strictly matter, but B → B′ (parent first, child second) keeps the visual fence opener tag flipped first.

Child reconfigure dispatch shape (RESEARCH §5, D-12):
```
const childView = this.childEditorRegistry.get(file.path);
if (childView) {
  const indentOverride = this.settings.getIndentSizeOverride();
  childView.dispatch({
    effects: languageCompartment.reconfigure(
      buildLanguageExtensions(newSlug, indentOverride)
    ),
    userEvent: 'leetcode.lang-switch',
  });
}
```

Key dispatch properties (RESEARCH §5):
- `effects` only — NO `changes`, NO `selection` (Compartment.reconfigure is StateEffect-based)
- `userEvent: 'leetcode.lang-switch'` (CLAUDE.md convention; matches the parent-side annotation)
- Cursor preserved (no document change)
- No remount: `Compartment.reconfigure` updates state in-place; the EditorView stays mounted
- No echo: child sync extension's `if (!update.docChanged) return` guard at childEditorSync.ts:89 skips effects-only transactions (RESEARCH §6 — VERIFIED)
- nestedEditorExtension fast-path: dispatch goes to CHILD, not parent; parent StateField never sees it; no widget rebuild (RESEARCH §6)

ChildEditorRegistry interface (src/main/childEditorRegistry.ts):
```
get(key: string): EditorView | undefined
```
Returns `undefined` when no child is registered for that file path. Code branch must handle `undefined` silently — many lc-slug notes may not have an open child editor when the user switches via the chevron from a different leaf or the registry was evicted.

Plugin instance access:

- `this.childEditorRegistry` is the registry instance (per Phase 13 wiring)
- `this.settings` is the SettingsStore instance — `this.settings.getIndentSizeOverride()` returns 'auto' | 2 | 4 | 8
