<objective>
Replace the bare `indentWithTab` keymap entry in the child editor with a custom `Tab` command that branches on cursor position (CONTEXT D-11): line-start (or selection that spans line-starts) indents the line(s); mid-line inserts indentUnit characters at the cursor. Multi-line selection always indents as a single undo step (CONTEXT D-12 — Phase 15 INDENT-03 invariant preserved). Shift-Tab dedents regardless of position.

Purpose: Phase 16 UAT Test 15 surfaced the carry-over: "Tab indents the whole line regardless of cursor position; ideal CM6 behavior would be 'insert tab at cursor mid-line, indent line at line-start'." This plan implements that behavior using the documented `insertTab` + `indentMore` commands from `@codemirror/commands` (no hand-rolled indent calculation per RESEARCH §Don't Hand-Roll).

Output: `customTabCommand` and `customShiftTabCommand` defined in `src/main/childEditorFactory.ts`, bound first in the main keymap, with a vitest spec covering line-start / mid-line / multi-line-selection / single-undo / indentUnit-aware cases.
</objective>
