**Issue 5 — Vim Insert-mode cursor render intermittent:** After pressing `i` to enter Insert mode, the caret cursor sometimes does NOT render — no cursor visible, no blink. Once the user types any letter, the cursor appears and behavior normalizes. Pattern is non-deterministic.

Root cause analysis (per 17-UAT.md Test 17 hypothesis):
@replit/codemirror-vim's Insert-mode cursor relies on CodeMirror dispatching a measure pass after the modal state machine transitions Normal → Insert. The child editor's `Decoration.widget({block:true})` mount may interact with vim's transition timing — the cursor DOM element's visibility/opacity toggles before the next animation frame, missing the paint cycle. The first user keystroke triggers a transaction → forces the missing measure/draw.

Fix: Add explicit CSS rules under `.lc-nested-editor` scope that force `.cm-cursor` and `.cm-fat-cursor` to be visible (`opacity: 1; visibility: visible;`) and apply the standard CM6 cursor blink animation. This makes the cursor visibility independent of the vim-package's measure-pass timing.

**Issue 6 — Vim mode panel (`.cm-vim-panel`) not wired:** The `@replit/codemirror-vim` package ships an optional status panel showing `-- NORMAL --` / `-- INSERT --` mode indicators. Plan 17-06's call `vim()` (no options) does NOT include the panel; the package's signature is `vim(options = { status?: boolean })` and the panel is gated on `options.status === true` (verified at `node_modules/@replit/codemirror-vim/dist/index.js:8767`). Pass `{ status: true }` to enable.

Root cause: Plan 17-06 used `vim()` (no options) — `options.status` defaulted to falsy → `showPanel(statusPanel)` not added to the extension array → no panel renders.

Fix: Replace `vim()` with `vim({ status: true })` at `src/main/childEditorFactory.ts:261`. Add styling for `.cm-vim-panel` in styles.css under the `.lc-nested-editor` scope so the panel integrates visually.

Purpose: Closes 17-UAT.md Issues 5 + 6. Both fixes are cosmetic-only and entirely contained in the vim mount block + CSS. No new userEvent, no new module, no new dispatch. Plan 17-06's D-18 conditional contract preserved (vim still gated on `getConfig('vimMode')`).

Output: Updated vim() call with status option; CSS rules forcing cursor visibility + styling the vim panel; source-level + behavioral tests covering both fixes.
