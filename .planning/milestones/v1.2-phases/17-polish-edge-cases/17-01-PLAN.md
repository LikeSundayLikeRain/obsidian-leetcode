<objective>
Refactor the Reset code write-path to dispatch through the child editor's CM6 instance instead of the parent, restoring the Phase 15 D-05 cm-z scope isolation invariant for Reset (CONTEXT D-03..D-06). Currently `src/main.ts:2791-2803` dispatches on the parent EditorView with `userEvent: 'leetcode.reset'`, which lands the change in the parent's undo stack — Cmd-Z then inserts the prior solution body into the `## Notes` section (user screenshot evidence). After this plan, Reset behaves like a normal child edit: the child gets the undo entry, and the existing `createChildSyncExtension` mirror at `src/main/childEditorSync.ts:107-115` propagates the change to the parent with `addToHistory.of(false)`.

Purpose: Close the Phase 16 carry-over Reset undo regression that breaks the cm-z scope isolation invariant (CONTEXT D-03 architectural). Establishes the canonical plugin write-path pattern (CONTEXT D-05) that future write paths (Copy to Code audit deferred) will follow.

Output: `getDispatchHandle` in `src/main.ts` rewired to use `childEditorRegistry`, new userEvent `'leetcode.reset.child'`, regression test that fails on `main` today and passes after this refactor, CLAUDE.md `## Conventions` block updated to document the canonical pattern.
</objective>
