<objective>
Add line-numbers gating to the child editor following the established D-18 vim conditional-by-Obsidian-config pattern. User manual testing on 2026-05-23 (round 2 — post 17-07..17-11 gap-closure) surfaced LINENUM-01: no line-number gutter renders in the child editor regardless of Obsidian's global `showLineNumber` setting, and vim `:set nu` cannot help because there is no gutter wired in for vim to toggle. This plan makes the child editor honor `app.vault.getConfig('showLineNumber')` at mount, exactly mirroring the D-18 vim mount logic so the codebase stays internally consistent.

Purpose: The D-18 vim conditional is the canonical "read an Obsidian global config at child mount and conditionally spread an extension into the array" pattern (CONTEXT 17 Established Patterns: "Conditional extension loading pattern: D-18 vim is the first conditional-by-runtime-setting extension"). LINENUM-01 is the second instance of the same shape. Implementing it identically — same read site, same conditional spread, same remount-required reactivity contract — keeps the factory legible and prevents accidental drift toward bespoke listener-based config tracking.

Output: One conditional spread in `src/main/childEditorFactory.ts` adjacent to the existing vim conditional (lines 244-261), one new import line for `lineNumbers` (added to the existing `@codemirror/view` import block at lines 23-31, so the diff to import statements is one symbol), three new tests in `tests/main/childEditorFactory.test.ts`, one UAT entry for the gap-closure round.
</objective>
