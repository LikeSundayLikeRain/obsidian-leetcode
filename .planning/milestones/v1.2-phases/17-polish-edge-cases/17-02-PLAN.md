<objective>
Debug and fix the fence opener/closer auto-recovery regression surfaced 2026-05-23 (CONTEXT D-06b..D-06d): `repairFenceStructure` in `src/main/childEditorSync.ts:355` is not behaving correctly. The investigation must surface the root cause from the four CONTEXT D-06c hypotheses (a/b/c/d) BEFORE any fix is committed; the fix is whatever the debug session confirms, with a regression test that fails on `main` today and passes afterward.

Purpose: `repairFenceStructure` is the safety net that keeps the parent doc's fence syntactically intact when users edit fence boundaries. When it doesn't fire (or fires but doesn't propagate), child↔parent sync silently breaks — keystrokes get dropped, body offsets drift, and eventually the user sees a corrupted note. Restoring this safety net is a Wave-1 prerequisite for the Wave-2 edge-input UAT, because UAT scenarios (paste, IME) can damage fence boundaries.

Output: A debug doc enumerating the four hypotheses, instrumentation evidence, and the confirmed root cause; a fix scoped to the confirmed cause (most likely in `repairFenceStructure` itself or in `ECHO_PRONE_USER_EVENTS`); a regression test fixture covering damaged-opener / damaged-closer / both-damaged / intact / post-repair-sync cases.
