# Milestone v1.1: Contest, AI Coach, and Preview

**Status:** SHIPPED 2026-05-20

## Overview

Turn the v1.0 "solve and capture" plugin into a practice + coaching + AI-curated knowledge graph — adding contest training (including random virtual), AI debug & review, AI-maintained problem edges (including forward-looking edges to unsolved problems), and a non-destructive problem preview surface.

## Milestone Summary

**Decimal Phases:**

- Phase 08.1: Streaming transport fix + Bedrock provider (inserted after Phase 08 — streaming probe failed on Obsidian 1.10+ contextIsolation:true)
- Phase 08.2: Bedrock canonical default-chain (inserted after Phase 08.1 — credential resolution drift from canonical AWS chain)

**Key Decisions:**

- Bundle ceiling raised 500 KB → 1 MB (AI SDK lands on bundle graph as single unit; CJS-no-splitting makes dynamic import ineffective)
- Native `window.fetch()` primary streaming tier (electron.net.fetch probe fails on contextIsolation:true; obsidianFetch('stream') as fallback)
- Bedrock shipped as native 5th provider via `@ai-sdk/amazon-bedrock` (not deferred to v1.2 as originally planned)
- 22-pattern cluster taxonomy frozen at v1.1 ship (18 NeetCode + Prefix Sum + Monotonic Stack + Topological Sort + Union-Find)
- Look-ahead edges feature-flagged behind `featureFlags.lookAheadEdges`
- v1.0 → v1.1 `## Techniques` migration is lazy-on-AC (never batch on plugin load)
- AI streaming: 3-tier decision tree (native fetch → obsidianFetch('stream') → obsidianFetch('request') buffered)
- All AI vault writes via `app.vault.process` — never `cm.dispatch` or `vault.modify`

**Issues Resolved:**

- Streaming transport failure on Obsidian 1.10+ (Phase 08.1)
- AWS credential chain drift from canonical behavior (Phase 08.2)
- Verdict modal Close button layout issues (Phase 12)
- Contest scratch files polluting vault (Phase 12)
- Contest tab duplication on re-click (Phase 12)
- AI review hang in contest mode (Phase 12)

**Issues Deferred:**

- AIREV-06 (daily AI cost cap) — traceability shows Pending; implementation unclear
- LC Virtual Contest API integration (CONTEST-FUT-04) — register virtual session, contest-specific submit
- Native Bedrock SigV4 without credential_process (SSO refresh, assume-role chains)

**Technical Debt Incurred:**

- Bundle at 1.155 MB (above original 500 KB ceiling; within revised 1.2 MB ceiling)
- 10.1 "LC Virtual Contest API" phase listed in roadmap as Not Started (deferred backlog item)
