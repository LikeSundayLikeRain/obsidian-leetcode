<objective>
Close GAP-6: the currently-shipped `LeetCode.base` file exists but renders an empty Bases view in Obsidian 1.10+. Root cause is that our YAML schema (string-expression filters like `file.inFolder("LeetCode")` and `lc-id != null`) does not match Obsidian 1.10's actual Bases YAML schema — this is the A1 assumption-risk explicitly flagged as MEDIUM confidence in `02-RESEARCH.md`.

Approach (cannot be purely autonomous):
1. **Human checkpoint (reverse-engineering):** The user creates a working Bases file in the Obsidian UI (right-click folder → New base → add filters + table view sorted by `lc-id` DESC), copies the resulting YAML verbatim, and pastes it into the checkpoint response. We cannot guess the schema — the researcher already flagged it as MEDIUM confidence and the UAT confirmed the guess was wrong.
2. **Autonomous update:** Substitute the user-supplied YAML into `leetcodeBaseYaml()`, parameterising the folder name.
3. **Migration aid for existing installs (D-18-compatible):** Add a v0.1.0 signature detector + a one-time Notice on plugin load: "LeetCode.base may need to be regenerated. Delete it to get the updated view." Never auto-delete, never auto-overwrite — user must act.
4. **Human-verify checkpoint:** User deletes their old LeetCode.base, re-opens a problem, confirms the new .base renders a populated Bases table.

Purpose: The Bases view is the Phase 2 success-criterion-4 secondary sorted surface. It shipping empty turns a promised feature into an eyesore on every user's vault.

Output: A corrected `leetcodeBaseYaml` string matching Obsidian 1.10's actual schema; a pure `isLegacyLeetcodeBaseV010` detector; a one-time Notice wiring in `main.ts`; human-confirmed render in the target Obsidian install.

        * LeetCode.base schema. Shows once per install; user must manually delete
        * the file to regenerate — we never auto-delete or auto-overwrite (D-18).
        */
       private async checkLegacyLeetcodeBase(): Promise<void> {
         if (this.settings.hasShownLegacyBaseNotice()) return;
         const folder = this.settings.getProblemsFolder().replace(/[\\/]+$/, '');
         const path = `${folder}/LeetCode.base`;
         const file = this.app.vault.getAbstractFileByPath(path);
         if (!file) return;   // no .base yet — nothing to migrate
         // TFile-like read. Use vault.read if available on the abstract file.
         let contents = '';
         try {
           contents = await this.app.vault.read(file as unknown as TFile);
         } catch {
           return;   // unreadable — treat as non-issue
         }
         if (!isLegacyLeetcodeBaseV010(contents)) return;
         // eslint-disable-next-line obsidianmd/ui/sentence-case -- UI-SPEC.md § Copywriting LOCKED: "LeetCode" is a proper-noun brand name

         await this.settings.markLegacyBaseNoticeShown();
