<objective>
Close GAP-2b (LC example blocks render as flat paragraphs) and GAP-2c (HTML `<sup>`/`<sub>` tags leak as literal HTML) by extending the turndown configuration in `src/notes/htmlToMarkdown.ts` with three custom rules:

1. **lc-sup** — converts `<sup>X</sup>` → `$^{X}$` (Obsidian math-mode caret form)
2. **lc-sub** — converts `<sub>X</sub>` → `$_{X}$` (Obsidian math-mode underscore form)
3. **lc-example-block** — detects LC's example pattern (a `<pre>` whose previous sibling is `<p><strong class="example">Example …:</strong></p>` OR a `<pre>` containing `<strong>Input:</strong>`) and emits a fenced code block with the `text` language hint

Remove the existing `service.keep(['sub', 'sup', ...])` — the custom rules replace the keep-through.

Purpose: NOTE-02 promises rendered Markdown under `## Problem`. Literal `<sup>2</sup>` text in the middle of constraint lines (e.g. `10<sup>4</sup>`) visually degrades every problem note. Example blocks rendered as flat paragraphs destroy LC's Input/Output/Explanation structure — the second-most-hit rendering in every problem.

Output: three additive turndown rules, extended unit + determinism tests, updated snapshot files for all three LC fixtures (two-sum, median, regex).
</objective>

**Detection heuristic for lc-example-block (recommended):**
- Target: element whose tag is `PRE` AND whose child is NOT a `<code class="language-*">` element

- Equivalently: filter matches `<pre>` whose `firstElementChild` is NOT a `CODE` with a `className` matching `/language-/`
- When matched: emit fenced block with language hint `text`, preserving the inner text (stripping `<strong>` wrappers — the Input:/Output: labels should remain in the output text, just without bold)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LC HTML → turndown input | LC content is untrusted HTML from the network; turndown converts to Markdown (string), which is then written to a file (never the DOM) — no XSS vector |
| turndown → vault file | htmlToMarkdown output is a string passed to Vault.process / Vault.create; Obsidian handles YAML + file-system writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.7-01 | Tampering | Malicious LC HTML injects `<sup>` with fence delimiters to break out of the math inline | mitigate | Rule trims + wraps in `$...$`; the inner text is still Markdown-literal but contained in a math span. Even if an attacker inserted `$^{}$ ...`, the outer `$...$` pair provides the boundary. Obsidian renders it as math, not as runaway math/text |
| T-02.7-02 | Tampering | Example-block rule misfires on `<pre><code class="language-python">` causing real code to be mis-fenced as `text` | mitigate | Filter function explicitly checks `firstElementChild.className` for `/language-/` and defers; Test 7 + lc-regex snapshot pin this behavior |
| T-02.7-03 | Information Disclosure | `textContent` extraction in example-block rule leaks hidden HTML attributes | accept | `textContent` by design returns only visible text — no attribute leakage. Even if an attribute did leak, output goes to a vault file owned by the user |
| T-02.7-04 | Denial of Service | Turndown hang on deeply nested `<sup>` / `<sub>` | accept | Turndown walks DOM in order; stack depth is bounded by LC HTML depth (shallow — LC problems don't nest superscripts). No timeouts needed |
| T-02.7-05 | Tampering | Non-deterministic rule output breaks re-open idempotence | mitigate | Determinism test extended (Test 9) with a combined sup + example-block fixture, 100-run byte equality assertion |
