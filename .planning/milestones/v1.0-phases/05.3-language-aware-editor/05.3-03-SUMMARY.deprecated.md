### Task 1 (commit `4f05515`) — Standalone bundle-size gate script

`scripts/check-bundle-size.sh` — **34 lines, mode 100755**:
- `#!/usr/bin/env bash` + `set -eo pipefail` (PATTERNS §"Bash Build Gate Structure").
- `ROOT="$(cd "$(dirname "$0")/.." && pwd)"` + `MAIN="$ROOT/main.js"` — locates `main.js` from the repo root via the script's own `dirname`, so the script works from any cwd.
- Missing-file branch: `BUNDLE CHECK FAIL: main.js missing — run 'npm run build' first` → stderr → exit 1.
- `MAIN_BYTES=$(wc -c < "$MAIN" | tr -d ' ')` + `MAIN_KB=$((MAIN_BYTES / 1024))`.
- `LIMIT_BYTES=$((700 * 1024))` (D-09 cap) + `WARN_BYTES=$((600 * 1024))` (D-09 soft threshold).
- `>$LIMIT_BYTES` → `BUNDLE CHECK FAIL: main.js is N KB (>700 KB ceiling; D-09 escalation required)` → stderr → exit 1.
- `>$WARN_BYTES` → `BUNDLE CHECK WARN: main.js is N KB (>600 KB soft warning; ceiling 700 KB)` → stdout (no exit).
- Always trailing `BUNDLE CHECK OK: main.js N KB within budget` + exit 0 for any non-fail outcome.
- Pure bash + coreutils only (no `jq`, `node`, or other external tools per `<behavior>` constraint).
