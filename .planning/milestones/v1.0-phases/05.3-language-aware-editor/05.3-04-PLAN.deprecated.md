    ## Section A — Supported languages in Live Preview (core behavior)

    **Python (python3 slug):**
    1. Open an lc-slug Python note in Live Preview. Click inside the `## Code` fence.
    2. Verify syntax highlighting: keywords (`def`, `class`, `return`, `if`, `else`, `for`, `while`, `import`, `from`) render in the theme's keyword color; strings in string color; `# comment` lines in comment color.
    3. Type `def foo():` then Enter. Next line must indent 4 spaces (or the user's configured Editor→Tab indent width).
    4. Continue: type `return 1` then Enter. Cursor stays at same indent.
    5. Type `else:` at start of a new line. It should dedent relative to its matching `if:` block (lang-python `indentNodeProp` handles this).
    6. Type `(` — matching `)` auto-inserted. Same for `[`, `{`, `"`, `'`.

    **Java:**
    7. Open an lc-slug Java note (create one by fetching a problem with Java set as the default language, or edit an existing lc-slug note's fence tag to `java`). Verify keyword coloring on `class`, `public`, `static`, `return`.
    8. Type `public class Solution {` then Enter. Next line indents one level.
    9. Type `}` at start of a new line. Dedents to match the opening `{`.
    10. Type `(` → matching `)` inserted. Type `"` → matching `"` inserted.

    **C++ (cpp slug) and C (c slug — same parser):**
    11. Open an lc-slug note with a `cpp` fence. Type `#include <vector>` — `#include` should highlight as a preprocessor directive.
    12. Type `int main() {` then Enter → indent; `}` dedents.
    13. Open an lc-slug note with a `c` fence. Confirm same C-family indent behavior (shared `@codemirror/lang-cpp` grammar per D-05).

    **JavaScript / TypeScript:**
    14. `javascript` fence: type `const arr = [1, 2, 3];` — brackets highlighted, numbers colored.
    15. `javascript` fence: type `function f() {` then Enter → indent; `}` dedents.
    16. `typescript` fence: type `interface Foo { bar: number; }` — `interface` and `number` colored as TS types (`lang-javascript({ typescript: true })` path).

    **Go (golang slug):**
    17. Open an lc-slug note with a `golang` fence. Type `func main() {` then Enter → indent; `}` dedents.
    18. Verify Go keywords (`func`, `package`, `import`, `return`, `if`, `for`, `var`) colored distinctly.

    **Rust:**
    19. Open an lc-slug note with a `rust` fence. Type `fn main() {` then Enter → indent.
    20. Type `let x: &'a str = "hi";` — lifetime `'a` colored distinctly from the string literal.

    ## Section B — Unsupported languages (D-06 whitespace-copy fallback)

    21. Open an lc-slug note and change the fence tag to `csharp`. Observe: fence content renders as plain monospace (no keyword coloring).
    22. Inside the `csharp` fence, type `public class Foo` Enter. Next line inherits the leading whitespace from the previous line (zero here → new line starts at column 0).
    23. Indent 4 spaces + type some text Enter. Next line has 4 leading spaces (whitespace-copy works for non-zero indents).
    24. Type `{` in the csharp fence. **NO** matching `}` should be inserted (D-06: fallback is whitespace-copy ONLY; no bracket auto-close for unsupported).
    25. Repeat 21–24 for `ruby` and `mysql` fences. All must fall back identically.
    26. **No crash. No Notice. No disabled editor.** Editor continues to accept input. If ANY of these conditions fails, mark the entire plan FAILED and escalate.
