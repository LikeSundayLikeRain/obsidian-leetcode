## Background

The original Phase 5.3 plan (D-10 markdown-codeLanguages augmentation, D-11 per-view Compartment swap as Plan B) failed at the D-14 live-smoke checkpoint. Implementation built `languagePackRegistry`, `whitespaceCopyIndent`, `codeFenceLanguageExtension` and wired into `src/main.ts` Step 6i. All 53 unit tests passed. Initial setTimeout-defer fix resolved an "EditorView.update calls not allowed" error.

**The empirical failure:** dispatch fires successfully (verified via console logs `swap() loaded pack=python` → `swap() dispatching reconfigure` → `swap() dispatch OK`), but no syntax highlighting and no language-aware indent appear inside the fence. Root cause: Obsidian's `lang-markdown` nested-parser owns the fence sub-tree; CM6 routes language data per syntax-tree node, not per outer compartment.

A research agent investigated alternatives:
- **Path A — mutate Obsidian's internal `codeLanguages` map:** undocumented private API, no public access (RESEARCH §Pitfall 1).
- **Path B — nested EditorView per fence:** 2–3 weeks (bidirectional sync, cursor bridging, undo merging, Live-Preview re-render storms).
- **Path C — keymap-only intercept:** drop highlighting; ship indent-only behavior.

User proposed a fourth path: write-time fence-tag remap + chevron dropdown widget. Markdown's existing nested-parser does the highlighting work natively — we just need fence tags to be markdown-recognized.

## Three Investigation Reports

### 1. Frontmatter usage map
**Finding:** `lc-runtime-ms`, `lc-memory-mb` are write-only (display fetches fresh from LC GraphQL). `lc-language` is read at note open (NoteWriter.ts) for template input. **Verdict:** drop runtime+memory; keep lc-language.

### 2. Fence-tag remap table
**Finding:** Only 2 LC slugs need write-time remapping (`python3 → python`, `golang → go`). Java, cpp, javascript, typescript, rust, c (→cpp) work natively. Unsupported slugs render as plain monospace.

### 3. Chevron UI placement
**Finding:** Recommendation: inline with Phase 5.1's Run/Submit button row. Reuses `.leetcode-code-actions` flex container. User refined to **left-aligned chevron** and **Edit-Mode only**.
