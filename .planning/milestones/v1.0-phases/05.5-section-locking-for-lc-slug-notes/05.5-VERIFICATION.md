### UAT-Driven Gap Resolution

**G1 — Chevron dropdown staleness (RESOLVED)**

Symptom: chevron label and dropdown `is-current` highlight remained stale after language switch until next keystroke.

Fix verified in `src/main/codeActionsEditorExtension.ts`:
- `languageRefreshEffect` declared at line 112 as `StateEffect.define<string | undefined>()` — carries `newSlug` payload directly, bypassing the metadataCache race.
- `buildDecorations` accepts `overrideSlug?: string` parameter (line 241); overrideSlug wins over the cache read (lines 264-272).
- `StateField.update` passes the effect's slug through to `buildDecorations` as `overrideSlug` (lines 376-387).
- `switchFenceLanguage` in `src/main.ts` dispatches changes + `languageRefreshEffect` in one transaction.

**G2 — Visual-dim replaced by heading-marker hide (RESOLVED)**

The original D-04 visual-dim (`background: var(--background-secondary)`) was rejected during UAT. Replaced with `Decoration.line` + CSS `display:none` on `.cm-formatting-header` tokens.

Fix verified in `styles.css` lines 920-935:
```css
.cm-editor .leetcode-locked-heading-line .cm-formatting-header,
.cm-editor .leetcode-locked-heading-line .cm-formatting-header-2 {
  display: none;
}
```
`buildLockedDecorations` in `sectionLockExtension.ts` emits `Decoration.line({ class: 'leetcode-locked-heading-line' })` at each canonical heading's `line.from` (lines 284-301).

**G3 — Cursor navigation snap-out (RESOLVED)**

Symptom: cursor could land at locked heading positions via click and column-step arrows; ping-pong between adjacent locked clusters; Up from fence body teleported to frontmatter pocket.

Fix verified in `sectionLockExtension.ts`:
- `EditorView.atomicRanges` removed; replaced by `EditorState.transactionFilter` (lines 435-502).
- Pure helpers `mergeLockedRanges` (line 211) and `computeSnapTarget` (line 246) extracted and exported.
- `mergeLockedRanges` coalesces adjacent/overlapping clusters so ## Problem body end == ## Code heading start merges into one cluster (prevents ping-pong).
- `computeSnapTarget` three-branch decision tree: forward motion → escape to `lockTo`; backward motion with editable backTarget → escape to `lockFrom - 1`; backward into unreachable pocket or first-cluster → stay put (`prevHead`).
- Gate 0 in `changeFilter` (lines 357-363): only suppresses transactions with known user-input userEvents (`input.*`, `delete.*`, `undo`, `redo`); programmatic vault-sync dispatches (no userEvent) pass through — fixes vault corruption when `copyToCode` wrote the file.
- Selection bypass (lines 463-464): `isCollapsedCursor` check — when `head !== anchor` (user is shift-selecting), snap is skipped so locked text can be copied.

### Data-Flow Trace (Level 4)

The sectionLockExtension is a pure CM6 filter/decoration extension — no external data store. Data flow is live:

1. `buildSectionLockExtension(plugin)` receives Plugin instance for `app.metadataCache` lookups.
2. On every transaction: `tr.startState.field(editorInfoField)` reads current file from CM6 state (live).
3. `plugin.app.metadataCache.getFileCache(file)?.frontmatter['lc-slug']` reads live frontmatter.
4. `computeLockedRanges(tr.startState)` scans `state.doc` directly (live document state).

Gate 0 fix ensures programmatic vault-sync transactions (no userEvent) flow through unchanged — no static fallbacks masking real data. Status: FLOWING.
