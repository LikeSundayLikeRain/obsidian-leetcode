## Warnings

### WR-01: Redundant `EditorState` type alias import
**File:** `src/main/sectionLockExtension.ts:41-47`
`EditorStateType` alias unused as distinct type. TypeScript distinguishes value/type-space naturally.
**Fix:** Drop alias, use `EditorState` directly for both value and type.

### WR-02: `buildAtomicRangeSet` and `buildLockedDecorations` near-duplicate
**File:** `src/main/sectionLockExtension.ts:188-203, 218-233`
Two near-identical functions differing only in CSS class string.
**Fix:** Extract `buildDecorationSetFromRanges(state, className)`.

### WR-03: Pitfall 7 (pasted `## Problem` in `## Notes`) undocumented in README
**File:** `README.md` Section Locking subsection
RESEARCH §Pitfall 7 explicitly recommended documenting this edge case. User-pasted snippets containing `## Problem` literally would trigger spurious locks.
**Fix:** Add edge-case paragraph to README; optionally tighten heading detection with `seen: Set<HeadingKind>` to skip duplicates.

### WR-04: No test covers Source Mode (`editorLivePreviewField === false`)
**File:** `tests/main/sectionLockExtension.test.ts`
Implementation correctly does NOT consult `editorLivePreviewField`, but no test exercises Source Mode path. A future regression that adds the gate accidentally would not be caught.
**Fix:** Add unit test setting `editorLivePreviewField: false` and asserting filter still suppresses.

### WR-05: Test stub `makeFakeTransaction.annotation` ignores kind argument
**File:** `tests/helpers/obsidian-stub.ts:222-233`
Returns `userEvent` for ANY annotation kind lookup. Future implementation calling `tr.annotation(SomeOtherAnnotation)` would be masked.
**Fix:** Branch on `_kind === Transaction.userEvent` and return undefined for other kinds.

### WR-06: `extractChangeFilterCallback` probes opaque CM6 facet shapes via field-name guessing
**File:** `tests/main/sectionLockExtension.test.ts:110-144` + integration test
Brittle to CM6 internal shape changes. Test file's own comment acknowledges this.
**Fix:** Export `evaluateChangeFilter(plugin, tr)` as a named function from `sectionLockExtension.ts`; tests import directly.

### WR-07: Stale comment in `buildAtomicRangeSet` references "future" feature that already shipped
**File:** `src/main/sectionLockExtension.ts:181-187`
Visual dim is now active; comment language is historical/aspirational.
**Fix:** Update to reflect shipped state.

### WR-08: Redundant `as HeadingHit` / `as number` casts
**File:** `src/main/sectionLockExtension.ts:132, 144, 192, 222, 224`
Casts strip `undefined` from `noUncheckedIndexedAccess` without explicit guards.
**Fix:** Either remove casts (if not strict) or add explicit `if (!cur) continue;` guards.
