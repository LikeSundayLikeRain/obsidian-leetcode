## Decisions Made

- **LOCKED_HEADINGS lives in `src/notes/NoteTemplate.ts`** (per RESEARCH Open Question 2 recommendation): co-locates the lock-policy export with the four heading-line constants so future readers find them together. Preserves Phase 2 D-03 SSoT invariant — the lock extension imports from one place, never duplicates strings.
- **`extractChangeFilterCallback` test helper is forward-compatible:** walks the Extension array probing each entry's `.value` / `.fn` / `.callback` slot for a callable. Wave 1 may either return `[changeFilter.of(cb), atomicRanges.of(...)]` (flat composition) OR expose a dedicated named export — either approach satisfies the test. Documented as a contract comment in the test file header.
- **D-07 verified by absence — no it-block:** per the plan's explicit instruction, the architectural guarantee that the lock applies only in Edit Mode is documented in a header comment pointing at `src/main/codeActionsPostProcessor.ts` (the Reading Mode path that renders via MarkdownPostProcessor and never invokes CM6 transaction filters).
- **Pure shape-adapter factories — no `EditorState.create`:** per RESEARCH §Pattern 2, filter-callback unit tests do not need a real EditorView. The factories return `as unknown as EditorState` / `as unknown as Transaction` casts; production code never sees this type. This matches Phase 5.1's inline `makeState` / `makeStateWithFile` style verbatim.
