1. HTTP 401 with JSON body { detail: "Authentication credentials were not provided." }
   - Fixture: tests/fixtures/lc-submissions/list-session-expired.json
   - Detection: `status === 401 && body && typeof body === 'object' && typeof (body as { detail?: unknown }).detail === 'string' && /authenticat/i.test((body as { detail: string }).detail)`
   - Simpler robust form: `status === 401` is itself sufficient because LC only returns 401 on auth failure for this surface.

2. HTTP 403 bare
   - No accompanying body shape required (LC's GraphQL auth-failure is status-only).
   - Detection: `status === 403`.

3. HTTP 200 with body.errors[] containing an auth-adjacent message
   - Detection: `status === 200 && body && typeof body === 'object'`
     `&& Array.isArray((body as { errors?: unknown }).errors)`
     `&& ((body as { errors: Array<{ message?: string }> }).errors).some((e) => /auth(enticat|oriz)/i.test(e?.message ?? ''))`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LC GraphQL response (JSON) → plugin process | Untrusted wire input crosses into plugin; parsed via narrow mappers that validate required fields before use |
| submission_id string → GraphQL Int variable | User-supplied via picker click (came from listSubmissionsForSlug row.id); must parse cleanly to a finite positive integer |
| terminal.status_runtime / status_memory (strings) → frontmatter numbers | parseInt/parseFloat may return NaN; must be filtered before write |
| ctx.file (TFile) → vault writes | Trusted (came from getActiveProblemContext which validates lc-slug) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | GraphQL submissionDetails response parsing | mitigate | Narrow mapper reads `body.data.submissionDetails` via typed access. If `body.data` is null or `submissionDetails` missing, throw a typed Error (`'detailForSubmission: empty GraphQL data'`) — picker renders inline error (D-06), never crashes. No `JSON.parse` on scraped strings (GraphQL already delivered JSON). |
| T-04-03-02 | Tampering | submission_id → GraphQL Int variable | mitigate | Regex guard `/^[0-9]+$/.test(id)` BEFORE building variables; reject with `Error('invalid submission id')`. Then `Number.parseInt(id, 10)`; reject if `!Number.isFinite(n) || n <= 0`. Guard fires before any network hop. |
| T-04-03-03 | Information Disclosure | LC response text / body in debug logs | mitigate | logger.debug already redacts LEETCODE_SESSION + csrftoken (Phase 1). Response text/body may contain cookies (unlikely for GraphQL response but defensive). Do NOT log full res.text — log only `res.status` + truncated `res.text.slice(0, 200)` for status>=400 errors. Do NOT log full GraphQL variable blocks (submissionId is fine; never log cookies). |
| T-04-03-04 | Tampering | terminal.status_runtime parseInt → lc-runtime-ms | mitigate | Guard: `const n = parseInt(raw, 10); return Number.isFinite(n) ? n : undefined;` → frontmatter write skipped on NaN per D-10. Same for parseFloat on status_memory. |
| T-04-03-05 | Tampering | terminal.lang → lc-language frontmatter | mitigate | Trust LC-returned langSlug (plugin wrote submission with this slug; round-trip). Defensive: if `typeof terminal.lang !== 'string'`, fall back to `settings.getDefaultLanguage()`. |
| T-04-03-06 | Elevation of Privilege | topicTag.slug → lc/{slug} tag name | accept | LC-controlled string; tags are user-visible only (no filesystem reach). Defensive: filter non-printable ASCII if slug contains `\n` or `\t` (unlikely; log-debug if detected). |
| T-04-03-07 | Denial of Service | KnowledgeGraphWriter.onAccepted hanging | mitigate | All three steps wrapped in try/catch at step boundaries. Step-1 failure → return (abandon secondary steps, frontmatter is primary). Step-2 failure → return (stubs become unresolved wikilinks but note still updated). Step-3 per-stub failure → continue loop (D-19). No step blocks >100ms typically (vault ops are synchronous in-memory). |
| T-04-03-08 | Information Disclosure | Cookie leak via submissionHistoryClient error path | mitigate | Phase 1 logger redaction pattern masks LEETCODE_SESSION/csrftoken before log output. Phase 4 adds no new logging of request payloads. Response-body logs are truncated to 200 chars. |
| T-04-03-09 | Spoofing | LC session expiry spoofed via body.errors[] | accept | Low risk — an attacker capable of injecting body.errors would already own the HTTPS transport (which requires MITM of the Obsidian/Electron cert-validated channel). Effect of a spoofed expiry is a forced re-login Notice — no data loss, no privilege escalation. |

```typescript
/**
 * Detect LC session expiry from a GraphQL response body (AUTH-04 — Phase 1 OWNED).
 * Primary signal (most reliable): `data === null`.
 * Secondary signal: error-message pattern.
 *
 * Overloads (D-30 for Phase 4 + backward-compat for Phase 1/3):
 *   isSessionExpired(resp: unknown): boolean                 — Phase 1/3 body-only check (GraphQL `data: null` + errors[])
 *   isSessionExpired(body: unknown, status: number): boolean — Phase 4 D-30 signal set:
 *     (a) status === 401 with JSON body { detail: "Authentication credentials were not provided." }
 *     (b) status === 403 bare (no body shape required)
 *     (c) status === 200 with body.errors[] containing a message matching /auth(enticat|oriz)/i
 *   (the single-arg form stays untouched for Phase 1/3 callers)
 */

export function isSessionExpired(body: unknown, status: number): boolean;
export function isSessionExpired(resp: unknown, status?: number): boolean {
  // Two-arg form — D-30 signal set.
  if (typeof status === 'number') {
    // Signal (b) — HTTP 403 bare.
    if (status === 403) return true;
    // Signal (a) — HTTP 401 + JSON detail body.
    if (status === 401) {
      if (resp && typeof resp === 'object') {
        const r = resp as { detail?: unknown };
        if (typeof r.detail === 'string' && /authenticat/i.test(r.detail)) return true;
      }
      // LC's submission-list 401 is always the auth case for this surface;
      // treat bare 401 as session expired even if the body shape shifts.
      return true;
    }
    // Signal (c) — HTTP 200 with body.errors[] auth message.
    if (status === 200 && resp && typeof resp === 'object') {
      const r = resp as { errors?: Array<{ message?: string }> };
      if (Array.isArray(r.errors)) {
        return r.errors.some((e) => /auth(enticat|oriz)/i.test(e?.message ?? ''));
      }
    }
    return false;
  }
  // Single-arg form — existing Phase 1/3 body-only check (UNCHANGED).
  if (!resp || typeof resp !== 'object') return false;
  const r = resp as { data?: unknown; errors?: Array<{ message?: string }> };
  if (r.data === null) return true;
  if (!Array.isArray(r.errors)) return false;
  return r.errors.some((e) =>
    /logged in|authentication|CSRF|unauthori[sz]ed/i.test(e?.message ?? '')
```

```typescript
/**
 * On-AC knowledge-graph writer (D-08 entry point, D-09 three-step pipeline).
 *
 * Called from main.ts's submitFromActive AC branch (Plan 05). Orchestrates:
 *   1. processFrontMatter → 5 lc-* solve-time fields + union-merge lc/{topic-slug} tags (GRAPH-02, D-11)
 *   2. vault.process → ## Techniques region union-merge (GRAPH-03, D-13) — gated by opt-out (D-20)
 *   3. vault.create loop → stub technique notes (GRAPH-04, D-18) — gated by opt-out (D-20)
 *
 * D-23: only fires when classifyStatus(terminal.status_code).kind === 'ac'.
 * D-24: every AC re-fires all 3 steps — frontmatter reflects most recent AC.
 * D-25: no topic tags → step 1 fires (frontmatter), steps 2+3 skip.
 * D-19: step 3 per-stub failure is silent; ## Techniques write completes first.
 *
 * This class is a SINGLETON registered in main.ts (Plan 05). No captured
 * state between invocations — deps are injected via constructor.
 */
import type { App, TFile } from 'obsidian';
import { classifyStatus } from '../solve/statusMap';
import type { SubmitCheckResponse } from '../solve/types';
import type { DetailCacheEntry } from '../settings/SettingsStore';
import { applyFrontmatter, type NoteTemplateInput } from '../notes/NoteTemplate';
import { buildTechniqueStubBody, buildTechniqueFilename, LC_TAG_PREFIX } from '../notes/NoteTemplate';
import { toIsoLocalTz } from './dateFormat';
import { mergeTechniquesSection } from './mergeTechniquesSection';
import { ensureTechniquesFolder, createStubIfMissing } from './StubNoteCreator';
import { logger } from '../shared/logger';

/** Minimal ProblemContext slice the writer needs (matches main.ts shape). */
export interface ProblemContext {
  file: TFile;
  slug: string;
  title: string;
  // Additional fields on main.ts's ProblemContext are unused here.
}

/** Dependencies (constructor-injected; structural DI for testability). */
export interface KnowledgeGraphWriterDeps {
  app: App;
  settings: {
    getProblemDetail(slug: string): DetailCacheEntry | null;
    getAutoBacklinksEnabled(): boolean;
    getProblemsFolder(): string;
    getTechniquesFolder(): string;
    getDefaultLanguage(): string;
  };
}

export class KnowledgeGraphWriter {
  constructor(private readonly deps: KnowledgeGraphWriterDeps) {}

  /** Top-level entry point. Called from main.ts AFTER verdict modal renders. */
  async onAccepted(
    ctx: ProblemContext,
    terminal: SubmitCheckResponse,
  ): Promise<void> {
    // D-23 gate: only fires on AC. classifyStatus uses kind === 'ac'.
    const info = classifyStatus(terminal.status_code);
    if (info.kind !== 'ac') return;

    const detail = this.deps.settings.getProblemDetail(ctx.slug);
    if (!detail) {
      logger.debug('graph.onAccepted: no detail cache for slug; writing solve-time frontmatter only', { slug: ctx.slug });
      // Still fire step 1 (5 GRAPH-02 fields) via the dedicated cache-miss path.
      // Skips topic-tag union + Techniques + stubs (no detail cache → no topic
      // tags available). Phase 2 D-11's on-open background-refresh populates
      // topicTags for next time; next AC fires the full pipeline.
      try {
        await this.writeSolveTimeFrontmatter(ctx, terminal);
      } catch (err) {
        logger.debug('graph.onAccepted: writeSolveTimeFrontmatter failed', err);
      }
      return;
    }

    const topicSlugs = detail.topicSlugs ?? [];
    const topicTags = detail.topicTags; // D-12 — may be undefined (Pitfall 10 backward-compat)

    // Step 1 — frontmatter (ALWAYS fires on AC, even under opt-out per D-20).
    try {
      await this.writeFrontmatter(ctx, terminal, detail, topicSlugs);
    } catch (err) {
      logger.debug('graph.onAccepted: applyFrontmatter failed', err);
      return;
    }

    // D-20 opt-out gate — skip body + stubs but KEEP the frontmatter tags (D-20 rationale).
    if (!this.deps.settings.getAutoBacklinksEnabled()) return;

    // D-25 edge case + Pitfall 10 backward-compat: no topicTags → skip steps 2+3.
    if (!Array.isArray(topicTags) || topicTags.length === 0) return;

    // Step 2 — ## Techniques region rewrite (pure transform via vault.process).
    try {
      await this.deps.app.vault.process(ctx.file, (current) =>
        mergeTechniquesSection(current, topicTags),
      );
    } catch (err) {
      logger.debug('graph.onAccepted: Techniques write failed', err);
      return;
    }

    // Step 3 — stub creation loop (D-19 non-atomic; per-stub failures silent).
    try {
      await ensureTechniquesFolder(this.deps.app, this.deps.settings.getTechniquesFolder());
      for (const tag of topicTags) {
        const filename = buildTechniqueFilename(tag.name);
        const path = `${this.deps.settings.getTechniquesFolder()}/${filename}`;
        const body = buildTechniqueStubBody(tag.slug, tag.name);
        try {
          await createStubIfMissing(this.deps.app, path, body);
        } catch (err) {
          // Per-stub failure — silent debug log, continue loop.
          logger.debug('graph.onAccepted: stub create failed', { path, err });
        }
      }
    } catch (err) {
      logger.debug('graph.onAccepted: stub loop failed', err);
    }
  }

  // ---- private ----

  private async writeFrontmatter(
    ctx: ProblemContext,
    terminal: SubmitCheckResponse,
    detail: DetailCacheEntry,
    topicSlugs: ReadonlyArray<string>,
  ): Promise<void> {
    const input = this.buildInput(ctx, terminal, detail, topicSlugs);
    await applyFrontmatter(this.deps.app, ctx.file, input);
  }

  /**
   * Actual write path when detail cache is missing.
   *
   * Writes the 5 GRAPH-02 solve-time frontmatter fields directly via
   * `app.fileManager.processFrontMatter` (NOT via applyFrontmatter, because
   * applyFrontmatter requires NoteTemplateInput which needs difficulty + url +
   * title from the cache). Skips topic-tag union-merge (no detail → no topic
   * tags), skips the Phase 2 GAP-2a difficulty-tag write (no detail.difficulty),
   * skips `## Techniques` body + stubs (no topic tags).
   *
   * The Phase 2 GAP-2a non-downgrade guard for `lc-status` is preserved INLINE
   * (copy-pasted from applyFrontmatter): existing 'accepted' is never clobbered,
   * but any other existing value is upgraded to 'accepted' — which IS the AC
   * upgrade path. Re-AC on an already-'accepted' note: fm.lc-status stays
   * 'accepted' (no downgrade concern because the value is already 'accepted').
   */
  private async writeSolveTimeFrontmatter(
    ctx: ProblemContext,
    terminal: SubmitCheckResponse,
  ): Promise<void> {
    const runtimeMs = this.parseRuntime(terminal.status_runtime);
    const memoryMb = this.parseMemory(terminal.status_memory);
    const lang = typeof terminal.lang === 'string' && terminal.lang.length > 0
      ? terminal.lang
      : this.deps.settings.getDefaultLanguage();
    const solvedDate = toIsoLocalTz(new Date());

    await this.deps.app.fileManager.processFrontMatter(ctx.file, (fm: Record<string, unknown>) => {
      // Phase 2 GAP-2a non-downgrade guard — replicated here because we're
      // bypassing applyFrontmatter. Only upgrade to 'accepted'; never downgrade.
      if (fm['lc-status'] !== 'accepted') {
        fm['lc-status'] = 'accepted';
      }
      fm['lc-solved-date'] = solvedDate;
      if (typeof runtimeMs === 'number' && Number.isFinite(runtimeMs)) {
        fm['lc-runtime-ms'] = runtimeMs;
      }
      if (typeof memoryMb === 'number' && Number.isFinite(memoryMb)) {
        fm['lc-memory-mb'] = memoryMb;
      }
      fm['lc-language'] = lang;
    });
  }

  private buildInput(
    ctx: ProblemContext,
    terminal: SubmitCheckResponse,
    detail: DetailCacheEntry,
    topicSlugs: ReadonlyArray<string>,
  ): NoteTemplateInput {
    const runtimeMs = this.parseRuntime(terminal.status_runtime);
    const memoryMb = this.parseMemory(terminal.status_memory);
    const lang = typeof terminal.lang === 'string' && terminal.lang.length > 0
      ? terminal.lang
      : this.deps.settings.getDefaultLanguage();

    // Difficulty tag from cached detail; topic slugs unioned per D-11.
    const pluginTags = [
      `${LC_TAG_PREFIX}${detail.difficulty.toLowerCase()}`,
      ...topicSlugs.map((s) => `${LC_TAG_PREFIX}${s}`),
    ];

    return {
      id: detail.id,
      slug: ctx.slug,
      title: detail.title,
      difficulty: detail.difficulty,
      url: detail.url,
      language: lang,
      pluginTags,
      statusOverride: 'accepted',
      solvedDate: toIsoLocalTz(new Date()),
      runtimeMs,
      memoryMb,
    };
  }

  private parseRuntime(raw: string | undefined): number | undefined {
    if (typeof raw !== 'string') return undefined;
    const n = parseInt(raw, 10);
    return Number.isFinite(n) ? n : undefined;
  }

  private parseMemory(raw: string | undefined): number | undefined {
    if (typeof raw !== 'string') return undefined;
    const n = parseFloat(raw);
    return Number.isFinite(n) ? n : undefined;
  }
}
```
