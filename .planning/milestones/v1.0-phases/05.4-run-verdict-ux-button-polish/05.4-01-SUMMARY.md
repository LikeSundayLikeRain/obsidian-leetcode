## runArity.ts API

| Function | Signature | Fallback Behavior |
| --- | --- | --- |
| `parseMetaData(raw)` | `(string \| undefined \| null) → MetaData \| null` | Returns null on undefined/empty/malformed JSON/missing-params/malformed-param-entry. Never throws. |
| `deriveArity(metaData, sampleTestCase)` | `(string \| undefined \| null, string \| undefined \| null) → number` | Priority: parseMetaData→params.length≥1, else sampleTestCase non-empty-line count≥1, else 1. Always positive int. |
| `splitInput(joined, arity)` | `(string \| undefined \| null, number) → string[]` | Empty/whitespace → []; arity≤0 → [trimmed-joined]; otherwise group every `arity` non-empty lines. |
| `joinCasesForRun(cases, _arity)` | `(string[] \| undefined \| null, number) → string` | Filters non-string and empty-trimmed; joins with `\n`. `_arity` reserved for future arity-aware shaping. |
| `splitOutput(arr, arity)` | `(string \| string[] \| undefined \| null, number) → string[]` | Always length `arity` when arity>0; pads short arrays with `''`; normalizes single-string to single-element array. Pitfall 1 protection. |

**1. [Rule 1 - Bug] Fixed strict-null-check tsc errors in test files**
- **Found during:** Task 3 verification (`npx tsc --noEmit` reported errors at pre-existing lines 158 in RunModal.test.ts and 524-525 in languageChevronWidget.test.ts)
- **Issue:** `mock.calls[0][N]` and `arr[0][N]` index-access patterns triggered `TS2532: Object is possibly 'undefined'` under strict null checks introduced by `noUncheckedIndexedAccess` (or equivalent strict mode). The plan's `<verify>` command fails if any of the three test files report errors.
- **Fix:** Changed `mock.calls[0][N]` → `mock.calls[0]![N]` and `scrollAdds[0][1]` → `scrollAdds[0]![1]` (3 line edits across 2 files). The non-null assertion is safe because the surrounding `expect(...).toHaveBeenCalledTimes(1)` / `.toBeGreaterThanOrEqual(1)` assertion guarantees the entry exists.
- **Files modified:** tests/solve/RunModal.test.ts, tests/main/languageChevronWidget.test.ts
- **Commit:** included in Task 3 commit `bf6f9c7`

**2. [Rule 1 - Bug] Fixed strict-null-check tsc error in src/solve/runArity.ts**
- **Found during:** Task 2 verification (`npx tsc --noEmit` reported error at line 180 of new runArity.ts)
- **Issue:** `out[i] = i < source.length ? source[i] : '';` — strict mode treats `source[i]` as `string | undefined` (because of `noUncheckedIndexedAccess`).
- **Fix:** Switched to `Array.from({ length: safeArity }, (_v, i) => i < source.length ? (source[i] ?? '') : '')` so the type narrows to `string` and the lint rule against `new Array(N)` (`@typescript-eslint/no-unsafe-assignment`) is also resolved in the same change.
- **Files modified:** src/solve/runArity.ts
- **Commit:** included in Task 2 commit `a3e87e6` (pre-commit fix; first commit attempt would have failed lint).
