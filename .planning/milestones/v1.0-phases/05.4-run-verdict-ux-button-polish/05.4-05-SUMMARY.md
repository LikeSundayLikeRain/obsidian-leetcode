## A2 resolution

The original A2 assumption was: "live LC interpret_solution returns matching-length per-case arrays". **A2 is RESOLVED but with a refinement:** LC returns arrays that are **at least** the case count, but may pad with trailing empty strings (observed: 3-case Run returned length-4 arrays). The renderer no longer relies on array length for arity — it prefers `total_testcases` → `compare_result.length` → padded array length. The A2 contract holds for the per-index alignment within the response (`code_answer[i]` vs `expected_code_answer[i]`), which is what the verdict path actually consumes.

Plan 01's `tests/solve/fixtures/run-multi-case.json` is still SYNTHETIC. Refreshing it with the live capture is a non-blocking follow-up — current regression tests cover the padded-array case explicitly via inline `code_answer` arrays, so the fixture file's content is no longer load-bearing.
