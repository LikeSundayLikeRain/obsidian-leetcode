## Decisions Made

1. **Silent-copy test is a sibling file, not a modification of the existing
   SubmissionDetailModal.test.ts.**
   The existing file exercises the pre-Wave-1 confirm path (`copy-to-code
   confirms overwrite`) which is Wave 1 Plan 05.2-04's responsibility to
   reshape. Modifying it mid-Wave-0 would pre-commit Wave 1 decisions. The
   new shell lives at `tests/graph/SubmissionDetailModal.silent-copy.test.ts`.

2. **Python3 highlighter test uses dynamic import path to bypass Vite's
   import-analysis transform.**
   Vite's transform refuses to resolve a literal path to a non-existent
   module (even inside `it.skip`). Routing through a runtime variable with
   `/* @vite-ignore */` keeps the vitest loader happy while
   `src/main/python3Highlighter.ts` does not yet exist. 05.2-05 creates the
   source file; the test unskips as-is.

3. **All shells assert end-state helper exports that Wave 1 plans must
   extract.**
   `stripAutoDefaults(rules)`, `computeFilterBadgeCount(filter)`,
   `evaluateRule(row, rule)`, `handleFileOpenForStarterCode(plugin, file)`,
   `resetCodeForActive(plugin, file)`, `resetCodeCheckCallback(...)`,
   `getRegisteredCommandIds()`. This is intentional ‚Äî exposing these as
   pure helpers makes Wave 1 directly testable without Obsidian lifecycle
   plumbing.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Sibling filename for SubmissionDetailModal silent-copy test**

- **Found during:** Task 2 (creating SubmissionDetailModal silent-copy shell)
- **Issue:** Plan's `files_modified` lists `tests/graph/SubmissionDetailModal.test.ts`,
  which already exists (224 lines) with D-04 Phase 4 + D-31 Phase 5 Plan 05
  assertions, including a `copy-to-code confirms overwrite` test that
  exercises the pre-Wave-1 confirm path Wave 1 Plan 05.2-04 will delete.
  Writing the new silent-copy shell into the same file would destroy those
  tests. Plan action guidance is silent on this collision.
- **Fix:** Created sibling file
  `tests/graph/SubmissionDetailModal.silent-copy.test.ts` containing the D-10
  silent-copy shell. Legacy file is untouched. 05.2-04 reshapes the legacy
  file when silent overwrite lands.
- **Files modified:** `tests/graph/SubmissionDetailModal.silent-copy.test.ts`
  (new); NOT `tests/graph/SubmissionDetailModal.test.ts`.
- **Commit:** `9cbfc90`

**2. [Rule 3 - Blocking] Dynamic import path for missing source module**

- **Found during:** Task 2 (first vitest load of python3Highlighter shell)
- **Issue:** `await import('../../src/main/python3Highlighter')` inside
  `it.skip` still triggers Vite's import-analysis transform at load time,
  emitting `Failed to resolve import` and failing the whole test file with
  "0 tests" instead of "2 skipped". Plan action wrote "test will fail at
  import resolution" ‚Äî but Plan done criterion requires `vitest must load
  them without TypeScript errors`. The transform-time failure is not a TS
  error; it's a Vite resolver error, and it takes down the whole file.
- **Fix:** Routed the import through a runtime string constant with a
  `/* @vite-ignore */` prologue comment. Test remains `it.skip`; source
  doesn't need to exist; vitest loader reports the file as
  `2 tests | 2 skipped` instead of `failed`.
- **Files modified:** `tests/main/python3Highlighter.test.ts`
- **Commit:** `9cbfc90`
