# Phase 5.2 Plan 04: Starter-code auto-insert + Reset + silent Copy-to-Code Summary

Delivered the starter-code + Copy-to-Code cluster from Phase 5.2: auto-insert on file-open (D-06), a new `Reset code` command replacing `Insert starter code` (D-05/D-07), and silent Copy-to-Code in SubmissionDetailModal (D-10) while preserving ConfirmOverwriteModal for Reset (D-11).

## Scope

- D-05 remove `Insert starter code` command from `src/main.ts`.
- D-06 add `workspace.on('file-open')` hook that retrofits starter code silently whenever a note with a valid `lc-slug` frontmatter is opened.
- D-07 add `Reset code` command that force-injects the starter template behind `ConfirmOverwriteModal` when a non-empty fence is present.
- D-10 collapse `SubmissionDetailModal.handleCopyToCode` to a direct `performCopy` — no confirm gate.
- D-11 keep `src/graph/ConfirmOverwriteModal.ts` (sole remaining caller is the Reset command flow).

### Files Created

- `src/main/fileOpenHook.ts` — `makeFileOpenHandler(deps)` factory. Gates on `isValidSlug` + `metadataCache.getFileCache`, calls injected `retrofit` with the cached detail, and swallows rejections via `.catch(() => undefined)` per D-09 silent-on-failure.
- `src/solve/resetCodeWithConfirm.ts` — `resetCodeWithConfirm(deps)` async helper. Reads current body via `vault.read`, gates destructive overwrite behind `deps.confirm()` only when `hasExistingCodeBlock(body)` is true, force-injects via `vault.process(file, forceInjectCodeSection)`, and fires `notify('Code reset to starter.')` on success.

### Files Modified

- `src/main.ts` — imports updated (`retrofit as retrofitStarterCode`, `resetCodeWithConfirm`, `makeFileOpenHandler`); the `insert-starter-code` addCommand block became `reset-code`; the private `insertStarterCodeForced` helper became `resetCode`, a thin wrapper around the extracted helper that opens `ConfirmOverwriteModal` via the same dynamic-import pattern used elsewhere. A new Step 6g registers the file-open hook via `registerEvent(workspace.on('file-open', ...))`.
- `src/graph/SubmissionDetailModal.ts` — `handleCopyToCode` collapses to `await this.performCopy()`; `askConfirm`, `readCurrentBody`, and the `confirmOverwriteForTest` field are deleted; `hasExistingCodeBlock` + any ConfirmOverwriteModal references are dropped from this module.
