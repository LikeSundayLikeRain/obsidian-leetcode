## Live-smoke fixes applied inline

Two gap-closures surfaced and resolved during this plan's execution:

1. **Item 4 — badge count after Reset:** FilterModal's `Reset` button calls `onOpen()` which re-runs `ensurePrepopulated()`, seeding blank Status/Difficulty/Topics rules (`values: []`). Those are no-op rules (match everything) but were counted by `computeFilterBadgeCount`. Fix: `computeFilterBadgeCount` now excludes rules with empty `values` arrays in addition to `__autoDefault`-marked rules. User also needed to clear the legacy stale filter from `data.json` in the Leetcode vault (backup kept at `.bak.5.2-item-4-fix`).

2. **Item 9 — python3 highlighting:** First implementation used `registerMarkdownCodeBlockProcessor('python3', ...)` which DID highlight Reading Mode but hijacked Live Preview — python3 fences rendered as read-only widgets with an "Edit block" button, breaking normal in-place editing. Replaced with the lighter Prism-alias approach: `Prism.languages.python3 = Prism.languages.python` at plugin onload, plus the classList-rewrite post-processor as a belt-and-suspenders fallback. Reading Mode highlights; Live Preview stays natively editable. Edit-Mode CM6 highlighting explicitly deferred to **Phase 5.3** (language packs).

## Scope Clarification
