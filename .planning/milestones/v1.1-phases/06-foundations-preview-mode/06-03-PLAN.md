### Tab-reuse function signature (RESEARCH §Pattern 2)

```
// src/preview/previewRouter.ts
export async function openOrReusePreview(plugin: LeetCodePlugin, slug: string): Promise<void>
```

Behavior:
1. `const existing = plugin.app.workspace.getLeavesOfType(PREVIEW_VIEW_TYPE);`
2. If `existing.length > 0 && existing[0]`: call `await existing[0].setViewState({ type: PREVIEW_VIEW_TYPE, active: true, state: { slug } })` then `await plugin.app.workspace.revealLeaf(existing[0])`.
3. Else: `const leaf = plugin.app.workspace.getLeaf('tab')`; `await leaf.setViewState({ type: PREVIEW_VIEW_TYPE, active: true, state: { slug } })`; `await plugin.app.workspace.revealLeaf(leaf)`.

`WorkspaceLeaf.openIfExtant` does NOT exist (RESEARCH §Open Q9, verified by grep on obsidian.d.ts) — `getLeavesOfType + setViewState` is the canonical pattern.

### Existing-note detection helper (RESEARCH §3, copy from `src/notes/NoteWriter.ts:218-225`)

```
// src/preview/previewExistingNote.ts
export interface ExistingNoteState {
  fileExists: boolean;
  file?: TFile;
  id?: number;
}

export function detectExistingNote(
  app: App,
  settings: { getProblemsFolder(): string; getProblemDetail(slug: string): DetailCacheEntry | null },
  slug: string,
): ExistingNoteState
```

Implementation chain:
1. `const cached = settings.getProblemDetail(slug);` — returns `null` if no cache entry → `{ fileExists: false }`.
2. `const path = buildNotePath(settings.getProblemsFolder(), cached.id, slug);` — from `src/notes/NoteTemplate.ts:137`.
3. `const f = app.vault.getAbstractFileByPath(path);` — returns `TAbstractFile | null`.
4. If `f instanceof TFile` → `{ fileExists: true, file: f, id: cached.id }`. Else → `{ fileExists: false, id: cached.id }`.

The helper is PURE — no side effects, no awaits, fast (~3 function calls). Importable from tests without standing up a full plugin.

### Cache-miss render contract (RESEARCH §Open Q6 + RESEARCH §A2)

In `ProblemPreviewView.renderForSlug(slug)`:
1. `const cached = this.plugin.settings.getProblemDetail(slug);` — synchronous read.
2. If `cached` exists → render header + body immediately.
3. Else (cache miss):
   a. Render loading state synchronously per UI-SPEC copy ("Loading problem…" / "Fetching {id}. {title} from LeetCode.") — note: when slug-only is known, render `Loading problem… / Fetching {slug} from LeetCode.` since title isn't cached yet.
   b. Call `await this.plugin.client.getProblemDetail(slug)` (existing throttled `LeetCodeClient`, no new HTTP). **NOTE per RESEARCH §A2: `LeetCodeClient.getProblemDetail` does NOT auto-persist into SettingsStore — the preview view is responsible for persisting the freshly-fetched detail itself.**
   c. On resolve with `fetched.content` non-empty: `const entry = toDetailCacheEntry(fetched); await this.plugin.settings.setProblemDetail(slug, entry);` then re-read via `this.plugin.settings.getProblemDetail(slug)` and render. **The `setProblemDetail` call is mandatory — without it, the next preview of the same slug would re-fetch instead of hitting the cache.**
   d. On reject (network / session / timeout / rate-limit): render error state per UI-SPEC ("Couldn't load problem" / "Check your internet connection and try again." / `[Retry]` button).
