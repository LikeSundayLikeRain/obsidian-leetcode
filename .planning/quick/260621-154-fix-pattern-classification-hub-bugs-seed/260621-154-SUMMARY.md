# Quick 260621-154: Fix Pattern Classification + Hub Bugs Summary

Made `normalizePatternName` seed-aware and capital-preserving (FIX 1, casing drift), and added `sanitizeHubFilename` applied at all three hub-write sites plus the VerdictModal read path so slash-bearing pattern names ("Heap / Priority Queue") get a real, reachable hub note instead of being silently orphaned by `vault.create` (FIX 2 / 2b, P0).

## What Changed

**FIX 1 — `src/graph/patternTaxonomy.ts` (`normalizePatternName`)**
- Trim + collapse whitespace; empty/whitespace-only input returns `''` (no throw).
- Case-insensitive match against in-module `SEED_PATTERNS` returns the seed VERBATIM — `'1-d dynamic programming' -> '1-D Dynamic Programming'`, `'union-find' -> 'Union-Find'` (no longer mangled by `.toLowerCase()` on the remainder).
- AI-invented names: capitalize first char of each word WITHOUT lowercasing the remainder (`word.charAt(0).toUpperCase() + word.slice(1)`), preserving interior caps (`"Mo's Algorithm"`, `'Heavy-Light Decomposition'`).
- Module stays import-pure (zero imports). Doc-comment examples updated to demonstrate seed canonicalization + interior-cap preservation.

**FIX 2 — `src/graph/ClusterHubWriter.ts` (`sanitizeHubFilename`)**
- New `export function sanitizeHubFilename(name)` below the private-helpers divider: replaces each char in `/ \ : * ? " < > |` AND ASCII control chars `\x00-\x1F` with a single space, collapses `\s+`, trims. Idempotent.
- Applied to the FILENAME segment at all three sites — `ensureHub`, `appendEntry`, `reconcile` — so all derive the identical path (`LeetCode/Patterns/Heap Priority Queue.md`). `buildHubNoteBody` still receives the unsanitized display name, so content keeps `# Heap / Priority Queue` + `pattern: "Heap / Priority Queue"`. The fixed `{problemsFolder}/Patterns` literal is NOT sanitized. `reconcile`'s grouping Map stays keyed on the display name.

**FIX 2b — `src/main.ts` (read-path alignment)**
- Added `sanitizeHubFilename` to the existing `./graph/ClusterHubWriter` import (line 193).
- `getPatternHubPath` (line ~3503) now interpolates `sanitizeHubFilename(normalizePatternName(p))` — the exact writer transform — so the VerdictModal pattern chip resolves to the created hub (no dangling link). VerdictModal.ts fallback literal left untouched (out of scope).

**Tests**
- New `tests/graph/patternTaxonomy.test.ts`: seed identity for all 40 seeds, case canonicalization, interior-cap preservation, idempotence, empty-string. (RED first: 9 failures for the right reasons → GREEN.)
- Extended `tests/graph/clusterHubWriter.test.ts`: `sanitizeHubFilename` char-stripping, slash-pattern path + content, ensureHub/appendEntry/reconcile same-path resolution, and the read/write transform-agreement lock `sanitizeHubFilename(normalizePatternName('Heap / Priority Queue')) === 'Heap Priority Queue'`. (RED first: 5 failures → GREEN.)
