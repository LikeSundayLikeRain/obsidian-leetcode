<objective>
Fix two pattern-classification / hub-write bugs in the AI Knowledge Graph subsystem, plus tests.

FIX 1 (P1, casing-drift): `normalizePatternName` in `src/graph/patternTaxonomy.ts` lowercases the remainder of every word, mangling interior capitals of seed names even when the AI emits them verbatim ("1-D Dynamic Programming" -> "1-d Dynamic Programming", "Union-Find" -> "Union-find"). Make it seed-aware and capital-preserving while staying pure and idempotent.

FIX 2 (P0, orphaned hub): a "/" in a pattern name (seed "Heap / Priority Queue", or any future AI-invented slash name) is a path separator, so `vault.create` silently fails (never-throw logger.debug) and those problems get NO hub note. Sanitize the hub FILENAME at all three hub-path sites in `src/graph/ClusterHubWriter.ts` while keeping the unsanitized DISPLAY name in the note CONTENT.

FIX 2b (P0 completion, nav-seam): FIX 2 changes the hub WRITE path to "Heap Priority Queue.md", but the READ path `src/main.ts:3503` `getPatternHubPath` still derives the slash path via `normalizePatternName(p)` only. The VerdictModal pattern chip (`src/solve/VerdictModal.ts:291-296`) calls `getPatternHubPath` then `openLinkText(hubPath)` — so without this, clicking the "Heap / Priority Queue" chip navigates to a path FIX 2 just deleted. Align the read path to derive the filename identically to the writer: `sanitizeHubFilename(normalizePatternName(p))`. This mismatch is INTRODUCED by FIX 2 and must close in the same change to actually deliver P0 (reachable hub, no dangling link).

Purpose: Pattern hubs are the spine of the knowledge-graph value prop; casing drift fragments hubs and a "/" in a name orphans an entire cluster. The chip link must resolve to the file the writer actually creates.
Output: Two corrected pure helpers + one read-path call-site alignment + updated/new vitest specs. NO vault files, NO frontmatter, NO buildKgPrompt.ts touched.
</objective>
