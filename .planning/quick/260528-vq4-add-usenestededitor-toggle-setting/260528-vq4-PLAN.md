<objective>
Add a user-facing toggle that disables the nested CM6 child-editor stack at plugin onload, for users who prefer Obsidian's native markdown editor over the v1.2 nested editor experience.

Purpose: Users with custom CodeMirror plugins, accessibility tools, or who hit edge cases in the nested editor need a clean opt-out. The default stays ON to preserve current behavior for every existing user.
Output: One persisted boolean (`useNestedEditor`, default true), one toggle row in the existing "Code editor" settings group, one Notice prompting reload on change, and three onload-time gates that skip the nested-editor extension + its vault.on('modify') repair trigger + its file-open repair hook when OFF.
</objective>
